# 前端设计方案与原型技能

## 概述

本技能专注于**设计决策**，而非代码实现。
- `tx-ui` 回答"怎么写代码"
- `frontend-design` 回答"应该做成什么样"

当需要为屯象OS设计新页面、优化现有交互、或做设计评审时使用本技能。

## 第一步：识别设计任务类型

| 任务类型 | 触发词 | 输出物 |
|---------|--------|--------|
| **新页面设计** | "设计XX页面"、"做个XX" | 信息架构 + 布局方案 + 交互说明 |
| **体验优化** | "优化XX"、"XX不好用" | 问题分析 + 改进方案 + 对比 |
| **竞品分析** | "参考XX"、"对标XX" | 竞品拆解 + 可借鉴点 + 适配方案 |
| **设计评审** | "看看XX怎么样"、"检查XX" | 问题清单 + 改进建议 + 优先级 |
| **交互流程** | "XX的流程"、"用户怎么操作" | 流程图 + 异常分支 + 边界情况 |
| **信息架构** | "XX放什么内容"、"页面结构" | 内容层级 + 模块划分 + 导航结构 |

## 第三步：设计分析框架

### 3.1 用户画像确认
```
角色：[店长/收银员/厨师/服务员/总部运营/消费者/加盟商]
场景：[站着操作/坐着办公/边走边看/厨房高温油烟/休闲浏览]
设备：[安卓POS 10寸/安卓平板 10寸/手机 6寸/PC 14+寸/iPad 11寸]
频率：[每天100+次/每天10次/每周几次/偶尔使用]
技能：[专业操作员/普通员工/非技术消费者]
```

### 3.2 信息架构设计
```
页面标题
├─ 核心操作区（用户70%时间在这里）
│   ├─ 主要信息
│   └─ 主要操作按钮
├─ 辅助信息区（需要时查看）
│   ├─ 次要信息/筛选
│   └─ 辅助操作
├─ Agent 智能区（AI 建议/预警）
│   └─ 不可忽略的 Agent 提示
└─ 状态/导航区
    ├─ 面包屑/返回
    └─ 状态指示器
```

### 3.3 交互设计原则

**Store终端（触控优先）：**
- 触控目标 ≥ 48×48px，关键操作 ≥ 64×64px
- 单手可达：核心操作在屏幕下半部
- 无 hover 依赖：所有信息默认可见
- 操作反馈 < 100ms：点击即有视觉响应
- 误操作可撤销：关键操作有二次确认
- 减少输入：选择 > 输入，扫码 > 手打

**Admin终端（效率优先）：**
- 键盘友好：Tab 切换、Enter 确认、Esc 取消
- 批量操作：多选 + 批量动作
- 快速筛选：搜索框 + 标签筛选 + 高级筛选
- 数据密度适中：单屏信息量大但不拥挤
- 可定制：列显示/隐藏、排序记忆

**MiniApp（拇指优先）：**
- 拇指热区：底部 Tab + 底部操作栏
- 最少步骤：核心流程 ≤ 3 步
- 加载感知：骨架屏 + 进度指示
- 离线友好：缓存 + 重试机制

### 3.4 中餐连锁特殊设计考量

| 场景 | 设计要点 |
|------|---------|
| 高峰期收银 | 最少点击完成开单，0延迟反馈，醒目的菜品分类色块 |
| 厨房出餐 | 大字体、高对比、工单自动排序、超时变色预警 |
| 服务员巡台 | 边走边看，单手操作，快速识别桌台状态 |
| 总部巡店 | 多门店数据对比，异常高亮，下钻到单店明细 |
| 顾客点餐 | 菜品图片为主，价格醒目，加购顺滑，支付快捷 |
| 宴席预订 | 套餐定制、时间线、客户档案、AI建议菜单 |

## 第四步：输出设计方案

### 输出格式

**方案文档结构：**
```
1. 设计目标（一句话）
2. 用户故事（As a [角色], I want [操作], so that [价值]）
3. 信息架构（层级结构）
4. 布局方案（ASCII wireframe 或描述）
5. 交互说明（核心流程 + 异常处理）
6. Agent 集成点（哪些位置嵌入 AI 能力）
7. 设计决策记录（为什么这样做，不那样做）
8. 技术约束（需要哪些 API、数据、组件）
```

### ASCII Wireframe 规范
```
┌──────────────────────────────────┐
│  Header / Navigation             │
├────────┬─────────────────────────┤
│        │                         │
│  Side  │    Main Content         │
│  Nav   │                         │
│        │                         │
│        ├─────────────────────────┤
│        │    Secondary Area       │
├────────┴─────────────────────────┤
│  Footer / Action Bar             │
└──────────────────────────────────┘
```

## 核心铁律

1. **设计方案必须指定目标终端** — 不存在"通用"设计
2. **每个方案必须有用户故事** — 不做没有用户价值的设计
3. **Agent 集成点必须标注** — 屯象OS 的核心差异化是 AI，每个页面都要考虑
4. **三条硬约束必须可视化** — 毛利底线（红色标注）、食安合规（警告色）、出餐超时（倒计时）
5. **不做美工** — 本技能输出的是设计决策和结构，不是像素级视觉稿
6. **设计决策必须有理由** — "因为XX，所以选择YY"，不是"我觉得这样好看"

## 设计审查清单

- [ ] 明确了目标终端和用户角色？
- [ ] 核心操作在 3 步内完成？
- [ ] 考虑了高峰期的极端使用场景？
- [ ] Agent 智能能力有集成点？
- [ ] 三条硬约束有视觉表达？
- [ ] 异常/空状态有设计？
- [ ] 与现有页面的导航关系清晰？
- [ ] 不与现有 tx-ui 组件规范冲突？
