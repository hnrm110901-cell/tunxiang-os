# Word 文档模板库

## 商业提案

```
封面
  标题：[提案名称]
  副标题：为[客户名]定制的[解决方案类型]
  日期 + 版本 + 机密标注

目录（自动生成占位）

1. 执行摘要（1页）
   - 一段话总结：问题→方案→价值
   - 3个关键数字

2. 客户现状分析（1-2页）
   - 当前系统架构图
   - 痛点列表（表格：痛点 | 影响 | 频率）
   - 数据佐证

3. 解决方案（2-3页）
   - 总体架构图
   - 核心功能矩阵（表格）
   - 与现有系统的对比

4. 实施方案（1-2页）
   - 阶段划分（甘特图/表格）
   - 里程碑
   - 资源需求

5. 投资回报（1页）
   - ROI 计算表
   - 成本对比（Before/After）
   - 回收期

6. 团队与支持（0.5页）
7. 下一步（0.5页）

附录（按需）
```

## 计划文档

```
封面

1. 背景与目标（1页）
   - 项目背景
   - 核心目标（SMART原则）
   - 成功标准

2. 现状评估（1-2页）
   - 当前状态
   - 差距分析（表格）
   - 风险因素

3. 实施计划（2-3页）
   - 阶段划分
   - 每阶段：目标 | 任务 | 产出 | 时间 | 负责人
   - 依赖关系

4. 资源规划（1页）
   - 人员
   - 硬件/软件
   - 预算

5. 风险管理（1页）
   - 风险矩阵（概率×影响）
   - 应对策略

6. 里程碑与验收（0.5页）
```

## 操作手册

```
封面
  标题：[系统/功能名称] 操作手册
  适用版本 + 更新日期

1. 系统概述（1页）
   - 功能简介
   - 适用角色
   - 前置条件

2. 快速上手（1-2页）
   - 登录/进入
   - 核心操作3步走
   - 截图示例

3. 详细操作（N页）
   每个功能模块：
   - 功能说明
   - 操作步骤（编号列表 + 截图）
   - 注意事项（提示框）
   - 常见问题

4. 故障排除（1页）
   - 常见问题 FAQ（表格：问题 | 原因 | 解决）

5. 附录
   - 快捷键
   - 术语表
```

## 分析报告

```
封面

执行摘要（1页）
  - 核心结论（3-5条）
  - 关键数据亮点

1. 数据概况（1页）
   - 数据范围/时间段
   - 数据来源
   - 样本量

2. 核心分析（3-5页）
   每个分析维度：
   - 数据图表
   - 数据解读
   - 对比（环比/同比/对标）
   - 洞察/结论

3. 问题与机会（1-2页）
   - 问题列表（按严重度排序）
   - 机会列表（按价值排序）

4. 建议方案（1-2页）
   - 短期行动（本周/本月）
   - 中期优化（本季度）
   - 长期规划

5. 附录
   - 详细数据表
   - 方法论说明
```

## 招商手册

```
封面（品牌视觉冲击力）

1. 品牌介绍（1页）
   - 品牌故事
   - 发展历程
   - 企业愿景

2. 市场分析（1-2页）
   - 行业规模与趋势
   - 目标客群
   - 竞争格局

3. 产品/门店模型（2-3页）
   - 标准店型
   - 菜品结构
   - 运营数据（日均/月均/年均）

4. 加盟政策（2页）
   - 加盟条件
   - 费用明细（表格）
   - 回报模型

5. 支持体系（1-2页）
   - 选址支持
   - 装修支持
   - 培训支持
   - 供应链支持
   - **智能系统支持（屯象OS）**

6. 成功案例（1-2页）
   - 案例数据
   - 加盟商证言

7. 联系方式 + 下一步
```

## 需求文档（PRD）

```
封面
  产品名称 + 版本 + 日期

1. 概述
   - 背景与动机
   - 目标用户
   - 核心价值主张

2. 用户故事
   - As a [角色], I want [功能], so that [价值]
   - 验收标准（Given/When/Then）

3. 功能需求
   每个功能模块：
   - 功能描述
   - 交互流程（流程图/步骤）
   - 业务规则
   - 数据要求
   - 异常处理

4. 非功能需求
   - 性能
   - 安全
   - 兼容性

5. 排期与优先级
   - MoSCoW 优先级
   - 里程碑

附录：原型图/线框图引用
```
