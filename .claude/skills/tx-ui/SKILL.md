# 屯象OS UI 智能设计与开发技能

## 概述

屯象OS有四类前端终端，每类技术栈完全不同。
**生成任何前端代码之前，必须先确定目标终端，然后读取对应的参考文件。**

## 第一步：识别目标终端

根据用户描述判断终端类型：

| 关键词 | 终端 | 参考文件 |
|--------|------|----------|
| 总部、后台、管理、报表、驾驶舱、菜品管理、会员管理、财务、权限、Agent管理面板 | **Admin** | `references/admin.md` |
| 收银、POS、点餐（门店端）、结账、开台、桌台、挂账、核销 | **Store-POS** | `references/store.md` |
| KDS、出餐、后厨、厨房、工单、超时 | **Store-KDS** | `references/store.md` |
| 服务员、手机端（门店）、催菜、加菜、店长看板、PWA | **Store-Crew** | `references/store.md` |
| 小程序、顾客端、扫码点餐、会员中心、大厨到家、企业订餐、微信、抖音 | **MiniApp** | `references/miniapp.md` |
| 数据大屏、展示大屏 | **Screen** | 预留，暂用Admin规范 |
| 不确定 | → **停下来问用户** | — |

## 第二步：读取参考文件

确定终端后，**必须读取**对应的参考文件，然后再开始编码：

- Admin终端 → 读取 `references/admin.md`
- Store任何终端 → 读取 `references/store.md`
- MiniApp终端 → 读取 `references/miniapp.md`
- 所有终端都要读取 → `references/tokens.md`（Design Token规范）

如果任务涉及跨终端（如"同一个菜品数据在后台管理和小程序都要展示"），读取所有相关参考文件。

## 第三步：设计决策

在编码之前，先做以下判断（在思考中完成，不需要输出给用户，除非用户要求讨论）：

1. **这个界面的核心用户动作是什么？** 不是"展示信息"，而是用户做什么动作（点击结账？滑动完成出餐？筛选报表？）
2. **用户的物理环境是什么？** 坐在办公室(Admin)？站在收银台(POS)？站在厨房(KDS)？边走边操作(Crew)？躺在沙发上(MiniApp)？
3. **最可能出错的交互是什么？** 对Admin是误删数据，对POS是点错菜品，对KDS是漏看工单，对MiniApp是支付失败
4. **这个页面需要Agent介入吗？** 折扣守护Agent的预警条、AI推荐菜品的展示区域、异常数据的标注

## 第四步：编码

严格按照对应参考文件中的技术栈、组件库、编码规则生成代码。

## 核心铁律（所有终端通用）

1. **品牌色统一** —— 主色 `#FF6B35`，从Design Token引用，不硬编码
2. **API调用统一** —— 所有终端都带 `X-Tenant-ID` header
3. **类型安全** —— 从 `shared/api-types/` 引用类型定义，不手写interface
4. **Agent预警不可忽略** —— Store终端的Agent预警必须固定在屏幕顶部
5. **三条硬约束可视化** —— 毛利率低于阈值用红色标注，临期食材用警告色，出餐超时用倒计时

## 自学习机制

当用户对生成的UI给出修改反馈时：

1. 记录用户偏好（如"按钮要再大一点"→ 更新Store终端的默认按钮尺寸认知）
2. 记录业务规则（如"折扣率超过30%要用红色"→ 更新Agent预警的阈值展示规则）
3. 记录布局偏好（如"KDS卡片要横向排列不要纵向"→ 更新KDS的默认布局认知）

这些学习记录应在当前会话中持续应用，并在用户确认后建议更新到参考文件中。

## 设计审查清单

代码生成后，自动检查：

**Admin终端：**
- [ ] 使用了Ant Design 5.x组件？
- [ ] 列表页用了ProTable？
- [ ] 表单页用了ProForm/ModalForm？
- [ ] ConfigProvider注入了屯象主题Token？
- [ ] 最小支持1280px宽度？

**Store终端：**
- [ ] 使用了TXTouch组件（不是Ant Design）？
- [ ] 所有点击区域 ≥ 48×48px？
- [ ] 最小字体 ≥ 16px？
- [ ] 没有使用Select下拉菜单？
- [ ] 没有使用hover作为唯一反馈？
- [ ] 关键操作有二次确认？
- [ ] Agent预警固定在顶部？

**MiniApp终端：**
- [ ] 使用了uni-app + Vue 3 `<script setup>`？
- [ ] 样式用rpx单位？
- [ ] 没有引入Ant Design/Naive UI/Element？
- [ ] API调用走txRequest封装？
- [ ] 首屏包体积合理？
