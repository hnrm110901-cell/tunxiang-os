## 三、V6.0 三大战略升级

### 战略一：Predictive Engine（预测引擎）— 对标Toast IQ + Fourth iQ

**核心理念**：用机器学习预测驱动排班、采购、备餐、营销四大决策

```
┌─────────────────────────────────────────────────────┐
│                  Predictive Engine                    │
│                                                       │
│  输入层：                                              │
│  ├─ POS历史交易（3年+）                                │
│  ├─ 天气API（7天预报）                                  │
│  ├─ 本地事件（节假日/展会/赛事）                         │
│  ├─ 平台评价情感（美团/大众点评）                        │
│  └─ 门店画像（商圈/客群/竞对）                           │
│                                                       │
│  预测层：                                              │
│  ├─ 客流预测（小时级，7天滚动）                          │
│  ├─ 菜品需求预测（SKU级，含天气影响）                    │
│  ├─ 营收预测（日/周/月）                                │
│  └─ 异常预测（设备故障/差评风险）                        │
│                                                       │
│  决策层：                                              │
│  ├─ 智能排班（客流→最优人员配置）                        │
│  ├─ 智能采购（需求→安全库存→自动下单）                   │
│  ├─ 智能备餐（预测→半成品提前准备量）                    │
│  └─ 智能营销（预测低谷→自动触发私域券）                  │
└─────────────────────────────────────────────────────┘
```

**技术实现路径：**
- Phase 1: 基于PostgreSQL + Python的统计预测（ARIMA/Prophet）
- Phase 2: Core ML边缘部署轻量级模型
- Phase 3: Claude API深度推理（异常解释+策略建议）

### 战略二：Conversational OS（对话式经营系统）— 对标Toast IQ Assistant

**核心理念**：老板/店长用说人话的方式经营餐厅

```
老板问："上周五芙蓉路店为什么毛利下降了？"

系统回答：
"芙蓉路店上周五毛利率32.1%（环比-4.2%），主要原因：
 1. 波士顿龙虾进货价涨18%（供应商蜀海报价变动）→ 建议切换备选供应商或临时调价
 2. 折扣券核销率异常高（桌均2.3张 vs 正常0.8张）→ 发现优惠券被内部传播
 3. 晚市空台率35%（vs 上周五15%）→ 附近有竞对新店开业

推荐操作：
 [一键调价] 龙虾系列涨价8%  
 [一键暂停] 问题优惠券批次
 [一键推送] 老客召回券给沉默7天会员"
```

**技术架构：**
- NLQ引擎（Natural Language Query）→ 解析意图 → 路由到对应Agent
- Agent执行查询 → 物化视图+实时计算 → 结构化结果
- Claude API生成自然语言叙事 → 附带可执行操作按钮
- 多入口：web-admin对话框 / 企业微信 / 移动端语音

### 战略三：Multi-Brand Fabric（多品牌经营织网）— 对标Byte by Yum! + 海底捞红石榴

**核心理念**：集团共享中台+品牌独立运营，新品牌7天上线

```
┌──────────────────────────────────────────────────┐
│              Multi-Brand Fabric                    │
│                                                    │
│  集团共享层（Shared）：                              │
│  ├─ 统一供应链（蜀海模式：集采+央厨+配送）           │
│  ├─ 统一会员池（Golden ID跨品牌识别）                │
│  ├─ 统一技术平台（一套代码多品牌运行）               │
│  ├─ 统一数据中台（跨品牌经营洞察）                   │
│  └─ 统一Agent池（共享AI能力）                       │
│                                                    │
│  品牌独立层（Isolated）：                            │
│  ├─ 独立菜单+定价+营销策略                           │
│  ├─ 独立门店运营标准（SOP）                          │
│  ├─ 独立P&L核算                                     │
│  ├─ 独立客户体验（小程序/App/品牌调性）              │

│  Day 4: 对接支付+外卖平台                            │
```
