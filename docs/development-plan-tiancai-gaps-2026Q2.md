# 屯象OS × 天财商龙差距补齐 — 开发计划（2026 Q2-Q3）

> **依据**：2026-04-06 对标天财商龙「餐饮7」V7.7.86.230 × 屯象OS 功能差距分析
> **定位**：本计划聚焦天财成熟但屯象缺失/偏薄的功能，不重复 `development-plan-yellow-orange-gaps-2026Q2.md` 已覆盖的 🟡/🟠 条目
> **原则**：以"最小可交付"为粒度拆解，后端优先，前端紧跟，每项必须有可验收标准
> **基线**：Round 107（v184 迁移 / 16前端 / 13服务 / 320路由 / ~6900 测试用例）

### 优先级说明

| 优先级 | 含义 | 不做的后果 |
|--------|------|-----------|
| **P0** | 影响现有客户基本交付 | 现有门店无法正常运营/对账 |
| **P1** | 影响新客户规模扩张 | 无法赢得正餐品牌连锁大客户 |
| **P2** | 提升产品竞争宽度 | 特定场景/客群无法覆盖 |
| **P3** | 差异化护城河 | 屯象AI优势无法持续拉开 |

## Sprint 1：P0 报表核账（2026-04 W2-W3）

> **目标**：补齐财务核账报表体系，让门店/总部可自助对账，不依赖人工导数

### TC-P0-05：日结监控看板

**背景**：天财有实时门店日结状态看板，屯象tx-ops已有日结路由但总部侧无聚合视图。连锁总部每天需要确认所有门店是否按时日结。

**涉及模块**：`tx-ops`（后端）/ `tx-analytics`（聚合）/ `web-admin`（前端）

| # | 任务 | 可验收标准 |
|---|------|-----------|
| 1 | `tx-ops`：新增 `GET /api/v1/ops/daily-settlement/monitor` — 返回所有门店当日日结状态（门店ID/名称/应结时间/实际完成时间/状态: 未开始/进行中/已完成/逾期）| 返回含5家模拟门店、状态字段完整 |
| 2 | `tx-ops`：写入 `settlement_status` 物化视图或聚合查询，支持按品牌/区域过滤 | 品牌维度筛选正确 |
| 3 | `tx-analytics`：日结完成率趋势（按周/月），逾期日结门店排行 | 数据与tx-ops一致 |
| 4 | `web-admin`：财务核账 → 日结监控页，实时表格（门店/状态/时间差/操作员），支持手动标记/备注 | 页面可访问，表格数据正确 |
| 5 | 预警推送：超过设定时间未日结 → 企微/短信通知（复用现有通知基础设施）| 模拟触发预警通知成功 |
| 6 | 测试 ≥ 4个用例：多门店状态聚合/逾期检测/品牌筛选/总部权限隔离 | pytest通过 |

**预估**：3-4天 / 新增迁移：无（复用现有表）

## Sprint 2：P0 门店专项（2026-04 W4 — 05 W1）

> **目标**：补齐天差地别的门店操作细节，确保高端正餐客户（如徐记海鲜）签约无障碍

### TC-P0-01：营业市别精细化

**背景**：天财把每日分成早市/午市/晚市/夜宵等多个市别，不同市别可以有独立菜谱、价格方案，报表也按市别统计。这是连锁正餐的标准运营方式。

**涉及模块**：`tx-trade` / `tx-analytics` / `web-pos` / `web-admin`

| # | 任务 | 可验收标准 |
|---|------|-----------|
| 1 | DB迁移：确认/完善 `market_sessions` 表（session_id/name/start_time/end_time/store_id/is_active），支持门店级自定义市别时段 | 迁移可重复执行，含RLS |
| 2 | `tx-trade`：开台接口写入当前市别（根据开台时间自动关联当前进行中市别）；`orders` 表增加 `market_session_id` 字段 | 开台时订单自动携带market_session_id |
| 3 | `tx-trade`：换市别触发换班流程（前一市别统计快照，新市别开始计数）| 换市别后营业数据按市别隔离 |
| 4 | `tx-analytics`：营业情况报表增加"按市别"维度（汇总/明细/对比），经营指标可按市别切片 | 同一天三个市别数据可分开查看 |
| 5 | `tx-menu`：菜谱方案支持绑定市别（早餐菜谱/午市菜谱）；开台时自动加载对应市别菜谱 | 切换市别后POS菜单同步变化 |
| 6 | `web-admin`：营业档案 → 门店营业市别配置页（CRUD：市别名称/时间段/绑定菜谱）| 配置页可访问，保存生效 |
| 7 | `web-pos`：用户桌面显示当前市别；换市别操作入口 | 市别标识显示正确 |
| 8 | 测试 ≥ 5个用例：市别自动判断/换市别/按市别报表/菜谱绑定/多门店独立市别 | pytest通过 |

**预估**：4-5天 / 新增迁移：v185_market_session_order_binding
