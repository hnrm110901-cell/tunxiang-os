# 屯象OS × 徐记海鲜 专项交付计划

> **定位**: 屯象OS 是面向集团化、多品牌、多区域、多收入渠道的中餐连锁经营操作系统 — 餐饮行业的 Palantir
> **标杆客户**: 徐记海鲜（高复杂直营正餐集团，23套系统待替换）
> **核心策略**: 以徐记海鲜交付倒逼平台能力建设，每个为徐记开发的功能都是平台通用能力

---

## 二、交付原则

### Palantir定位下的产品策略

1. **平台先于项目** — 为徐记开发的每个功能必须是平台通用能力，不做定制化
2. **数据先于功能** — 先打通数据流（POS→屯象→报表），再做业务功能
3. **替换而非并行** — 明确替换路径：先接管→再替换→最后退役旧系统
4. **集团管控优先** — 总部→区域→门店的管控链路是集团化的核心

### 徐记交付的四个里程碑

```
M1 (Week 1-4):  数据可见 — 总部能看到所有门店的经营数据
M2 (Week 5-8):  交易可控 — POS+KDS+收银全流程替换旧系统
M3 (Week 9-14): 会员可运营 — 会员体系+营销活动上线
M4 (Week 15-20): 供应链可管 — 采购→库存→成本→报表闭环
```

## 八、徐记海鲜样板成功的四个证明

完成M1-M4后，徐记海鲜样板可以证明:

1. **少亏折扣** — 折扣守护Agent + 稽核报表 → 每月减少X万异常折扣
2. **少亏损耗** — BOM成本引擎 + 盘点差异 → 食材损耗率降低X%
3. **更快翻台** — KDS调度 + 出餐超时预警 → 平均出餐时间缩短X分钟
4. **总部看清** — 60+报表 + 驾驶舱 → 替换23套系统中的核心报表需求

这四个数字就是屯象OS复制到下一个客户的核心卖点。
