# **一、先给总判断**





屯象OS最合理的战略，不是去做“全民餐饮 SaaS”，而是先占住一个更窄、但更有壁垒的心智：



**“中国高复杂直营正餐集团经营操作系统”**。



这个判断有三个依据：



第一，中国餐饮市场足够大，2024 年市场规模约 **5.6 万亿元**，连锁化率约 **22%–23%**，且仍有继续提升空间；这说明“连锁化 + 数字化 + 供应链化”仍在继续，不是没机会，而是必须选对切口。 



第二，徐记海鲜本身就是高价值样板。徐记官网与品牌资料显示，其目前有 **全国 60+ 家直营门店**，覆盖上海、深圳、武汉、长沙、西安、株洲等城市，且明确强调直营与中高档海鲜酒楼定位。 



第三，你现有方案本质上已经不是轻 SaaS，而是 **Apple 生态 + Web App + 边缘 AI + L0-L4 五层架构 + 八大业务域 + 旧系统 Adapter 替换** 的重体系。文档明确写了 Apple 全家桶、Mac mini 边缘中枢、WKWebView、八大产品域、旧系统分阶段退役、以及“前 3 客户深度定制、后续 80% 标准化”的路径。   



所以，这一版我直接给你三件事：



1. 目标客户分层
2. 竞品对位图
3. 3 年路线图





------





# **二、目标客户分层**







## **2.1 核心客户层：必须优先打透**







### **A 类客户：高复杂直营正餐集团**





**定义：**



- 直营为主
- 10 家以上门店
- 客单较高
- 有包厢、宴请、桌台调度
- 有称重、活鲜、复杂损耗、复杂供应链
- 总部有较强经营管控需求





**典型代表：**



- 徐记海鲜这一类海鲜/酒楼/宴请型品牌
- 中高端湘菜、粤菜、融合中餐的大店模型
- 多品牌直营集团





**为什么优先：**

因为这类客户最痛的，不是“有没有收银系统”，而是：



- 旧系统多，数据不通
- 门店流程复杂
- 总部难穿透
- 供应链和损耗管理难
- 财务和经营口径常常不一致





这恰好对应你方案里的强项：

L1 Ontology、六大实体、人财物溯源、八大域、Adapter 过渡替换。  





### **A 类客户的决策关键词**





他们买单看的不是“功能多”，而是：



- 能不能稳
- 能不能少亏
- 能不能管住折扣、损耗、挂账、出餐、库存
- 能不能逐步替换旧系统而不炸门店





------





## **2.2 次核心客户层：第二波复制对象**







### **B 类客户：标准中式正餐连锁**





**定义：**



- 5–30 家门店
- 正餐为主
- 有总部统一菜品、会员、报表、采购需求
- 供应链复杂度中等
- 桌台管理较复杂，但弱于宴请型酒楼





**适配度：高**

这类客户是徐记模板向外复制的第一层。

他们不一定需要“海鲜活鲜深能力”，但会需要：



- POS / KDS / 桌台
- 菜品与 BOM
- 会员/CDP
- 多门店经营报表
- 关键库存与财务稽核





文档中的菜品域、会员域、供应链域、财务域，正适合抽出标准中式正餐模板。 



------





## **2.3 机会客户层：有条件进入**







### **C 类客户：精品小店 / 新品牌试点店**





**定义：**



- 1–10 家门店
- 希望从一开始就建立标准化和数据化
- 对设计体验、总部穿透、小程序、会员经营有要求





**适配度：中**

可以做，但应该是 **Lite 版**，不能把徐记的大店逻辑原封不动搬过去。

你文档里已经有 L4 的 **Pro 大店 / Lite 小店 / 小程序 / 总部** 多形态前端设计，这个方向是对的。 



------





## **2.4 暂不优先客户层：战略上先放弃**







### **D 类客户：低价小店、夫妻店、极简快餐、重加盟低客单品牌**





**原因：**



- 更价格敏感
- 更依赖低成本硬件与快速部署
- 更容易被现有通用 SaaS 满足
- 屯象当前方案过重，Apple 硬件路线和深治理路线不适合先打这类市场





这里要非常克制。

你现在不是没有机会做这类客户，而是 **做他们并不能建立护城河**。

# **三、客户分层对应的产品包**







## **3.1 A 类客户：海鲜/酒楼/高复杂正餐 Pro**





**主卖点：**



- 复杂交易与桌台履约
- 海鲜称重/活鲜/损耗
- 包厢宴请管理
- 多系统整合替换
- 总部经营穿透
- 财务稽核与可追溯





**应该先卖的不是“全套替换”，而是这 4 个结果：**



- 折扣损失下降
- 出餐/翻台效率提升
- 食材损耗下降
- 总部报表与财务对齐





文档里把折扣守护、库存预警、财务稽核等 Agent 放入体系，方向正确。 



------





## **3.2 B 类客户：中式正餐 Standard**





**主卖点：**



- 标准 POS/KDS/桌台
- 菜品/BOM/沽清
- 会员/CDP
- 多门店经营看板
- 供应链与采购协同
- 总部统一菜单与权限





------





## **3.3 C 类客户：精品小店 Lite**





**主卖点：**



- 快速开店
- 菜单、点餐、收银、会员、小程序一体
- 总部远程可管
- 后续可平滑升级到 Standard / Pro

# **四、竞品对位图**





我先把话说透：



**屯象不应该与美团、客如云、二维火、哗啦啦比“谁模块更多”。**

因为主流玩家已经普遍覆盖了收银、点餐、会员、供应链、连锁管理等一体化能力。比如客如云官网明确展示了开台、预定、排队、点单、收银、供应链、会员营销、多端导流、开放生态、称重打印等能力。 



你真正该比的是：



**谁更懂“复杂直营正餐集团”的经营本体。**



------





## **4.1 竞品对位矩阵**



| **维度** | **美团/客如云等通用餐饮 SaaS**           | **屯象OS 应该占位**                          |
| -------- | ---------------------------------------- | -------------------------------------------- |
| 核心心智 | 一体化餐饮数字化工具                     | 复杂正餐集团经营 OS                          |
| 主要优势 | 标准化广覆盖、部署快、软硬件成熟、生态强 | 复杂流程治理、旧系统替换、总部穿透、经营闭环 |
| 擅长客户 | 小店、快餐、茶饮、中小连锁、通用场景     | 海鲜、酒楼、宴请、中高端正餐、多品牌直营     |
| 系统价值 | 提效、经营数字化、流量与点单协同         | 提效 + 少亏 + 可追责 + 统一经营语义          |
| 实施模式 | 标准产品为主                             | 样板客户沉淀模板，逐步标准化                 |
| 风险点   | 深复杂场景可能不够深                     | 架构过重、实施节奏过慢、硬件路线过窄         |

这个对位不是说屯象比他们“更高级”，而是要避开正面肉搏。



------





## **4.2 二维定位图**







### **横轴：场景复杂度**





从“标准化轻场景”到“高复杂经营场景”





### **纵轴：经营穿透深度**





从“工具系统”到“经营操作系统”



可以这样理解：



- **美团/客如云/二维火/哗啦啦**：

  大多在“中等复杂度 + 中等到较强经营数字化”的区域，很强，但更偏通用平台。 

- **屯象OS 理想定位**：

  应该放在“高复杂度 + 高经营穿透深度”的右上角。

- **徐记海鲜样板**：

  就是验证你能不能站到右上角的试金石。





------





## **4.3 屯象的真正差异化，不是 8 个模块，而是 4 个能力**







### **1. 复杂经营本体**





不是单个模块，而是统一 Customer / Dish / Store / Order / Ingredient / Employee 的经营语义。  





### **2. 旧系统替换能力**





不是重新卖一个新系统，而是能接管旧世界。

你的文档明确列了 POS、会员、供应链、业务中台、BI、巡店、HR、OA 的 Adapter 与退役阶段。 





### **3. 经营闭环能力**





发现问题，不只看报表，还要触发动作、跟踪执行、复盘。

这是 Agent OS 真正该证明的价值。 





### **4. 复杂直营模板能力**





不是服务“所有餐饮”，而是先把海鲜/酒楼/宴请/高端正餐这条线做成模板。



------





# **五、3 年路线图 V1**





这里我不按“功能越多越好”排，而按 **战略获胜顺序** 排。



------





## **Year 1：打透徐记，证明结果**







### **年度目标**





把屯象从“技术蓝图”变成“样板成果”。





### **核心任务**







#### **1）先打 P0 的四个必赢点**





- 折扣守护
- 订单/菜品毛利穿透
- 桌台/出餐效率看板
- 财务稽核初版





这些最容易形成 ROI 证明。

文档商业模式里也已经把 AI 增值服务作为营收层之一，但更重要的是先拿到可量化成果。 





#### **2）跑通交易最小闭环**





- 点单
- 收银
- 打印
- KDS
- 券核销
- 称重菜改价





因为这些是门店现场生死线。文档里外设桥接、打印服务、秤服务、Mac mini 中枢都已规划。  





#### **3）建立第一版主数据治理**





- 菜品主数据
- 原料主数据
- 会员身份统一
- 门店/组织/权限统一
- 经营口径统一







### **Year 1 成功标志**





不是签多少客户，而是：



- 徐记样板店稳定上线
- 至少拿出 3 个可量化经营改善指标
- 从“徐记项目”沉淀出第一版 **海鲜酒楼 Pro 模板**





------





## **Year 2：模板化复制，拿下同类客户**







### **年度目标**





从单一样板，进入同业态复制。





### **核心任务**







#### **1）形成 3 套模板**





- 海鲜酒楼 Pro
- 中式正餐 Standard
- 精品小店 Lite







#### **2）缩短交付周期**



你文档中写了 MDM 远程部署、新店上线 ≤ 半天 的目标。这个方向正确，但前提是模板化足够高。 





#### **3）推出“Apple 标准版 + 通用兼容版”**



Apple 全家桶对高端直营是加分项，但不能成为唯一入口。文档明确把 Apple 作为标准硬件战略，这适合样板期，但要为第二年复制预留兼容路线。 





#### **4）把 Agent 从“可演示”变成“可结算”**



重点不是上更多 Agent，而是让几个 Agent 真能证明经营结果：



- 折扣守护
- 库存预警
- 菜品优化
- 会员召回
- 财务异常预警







### **Year 2 成功标志**





- 拿下 3–8 家同业态客户
- 交付不再重度依赖定制
- 新客户 70%+ 用模板交付
- 形成可复制销售话术与实施手册





------





## **Year 3：建立赛道心智，做窄赛道第一**







### **年度目标**





不追求全行业最大，而是成为这条窄赛道里“最懂复杂正餐”的第一印象。





### **核心任务**







#### **1）建立品牌心智**



把对外叙事从：

“AI-Native 餐饮操作系统”

收敛成：

“复杂直营正餐集团经营操作系统”





#### **2）深化总部能力**





- 多品牌管理
- 区域/门店分层经营分析
- 供应链协同深化
- 财务结算深化
- 经营复盘自动化







#### **3）形成行业数据飞轮**



文档里提到“每个客户的数据让全网 Agent 更聪明”，但这件事只有在模板化和同类客户积累到一定规模后才会成立。 





### **Year 3 成功标志**





- 在海鲜/酒楼/中高端正餐赛道形成明确案例群
- 产品收入结构从项目费逐步转向“订阅 + 模块 + AI 结果费”
- 客户购买理由从“试试看”变成“这类门店就该用屯象”

# **六、建议你对外怎么讲**







## **6.1 对客户讲**





不要先讲：



- Ontology
- Agent OS
- Core ML
- L0-L4





先讲四个结果：



- 少亏折扣
- 少亏损耗
- 更快出餐与翻台
- 总部终于能看清门店





然后再讲：

“我们不是给你再加一个系统，而是帮你逐步接管一堆旧系统。”



------





## **6.2 对投资人/合作方讲**





核心叙事应该是：



**中国餐饮不是缺一个通用 SaaS，而是缺一个能处理高复杂直营正餐经营的操作系统。屯象从徐记海鲜这种高复杂样板切入，把最难场景打穿，再把能力模板化复制。**

最该先打的是：

**A 类：海鲜/酒楼/宴请/高复杂直营正餐集团**

**B 类：标准中式正餐连锁**

**C 类：精品小店 Lite**

- **Year 1：打透徐记，证明结果**

- **Year 3：建立复杂正餐赛道第一心智**
