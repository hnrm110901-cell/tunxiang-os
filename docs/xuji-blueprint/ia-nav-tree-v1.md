# **《屯象OS × 徐记海鲜 重构版 信息架构图（IA）+ 菜单导航树 V1》**





这版 IA 的核心原则，不再按“旧系统来源”组织，也不按“后台技术模块”组织，而是按 **角色任务 + 经营闭环 + 总部穿透** 来组织。底层仍然承接你现有方案里的五层架构、六大实体、八大业务域，以及“安卓 POS 管外设、Mac mini 管智能”的门店拓扑；但在前台表达上，要让门店员工、店长、区域、总部都能按自己的工作任务快速找到入口。  



同时，这版 IA 也严格围绕你现在最该证明的四个结果来收敛：**少亏折扣、少亏损耗、更快出餐与翻台、总部看清门店**。所以菜单设计不能“功能均衡铺开”，而要向交易、损耗、出餐、经营穿透倾斜。 



------





# **一、IA 设计总原则**







## **1.1 顶层不是“系统中心”，而是“经营中心”**





建议屯象OS × 徐记海鲜的顶层导航固定为 6 个经营中心：



1. 现场交易
2. 商品供应链
3. 日清日结
4. 经营分析
5. 客户经营
6. 组织与配置





这样做的原因是，原方案八大域虽然完整，但对门店和总部用户来说过于“系统化”。而徐记海鲜样板更适合把高频动作收束到几个经营中心里，再把低频配置收口到“组织与配置”。原八大域可在后台能力层保留，但 IA 层要更贴近使用场景。 





## **1.2 一级菜单按角色共识组织，二级菜单按动作闭环组织**





例如：



- 店员先找“开台/点菜/催菜/结账”
- 店长先找“营业巡航/异常/复盘”
- 总部先找“门店排行/毛利/整改追踪”
- 供应链先找“请购/收货/盘点/损耗”





这比按“交易域、财务域、运营域”更顺手。





## **1.3 Pro 大店与 Lite 小店共享一棵树，再做权限裁剪**





你现有方案已经明确 L4 是 Pro 大店 / Lite 小店 / 小程序 / 总部 Web 多形态前端，共享统一底座。IA 层最专业的做法不是做两棵完全不同的树，而是做一棵主树，通过角色、门店类型、业态进行显隐和裁剪。

# **六、导航命名规范建议**





为了让整体更专业统一，我建议命名统一遵循三条：



第一，一级菜单全部用“中心”结尾。

例如：现场交易中心、经营分析中心。



第二，二级菜单尽量用“名词 + 动作”或“经营对象 + 场景”。

例如：预订与到店、盘点与损耗、收银与结算。



第三，避免太技术化。

不要直接给一线用户看到：



- Ontology
- Agent OS
- CDP
- Adapter
- Core ML
- RLS





这些概念可以留在技术架构和总部配置里，对外和对店员都不应成为主导航语言。这个也和你文档里“先讲结果，不先讲技术名词”的建议一致。
