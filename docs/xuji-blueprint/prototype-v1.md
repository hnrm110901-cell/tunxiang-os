# **《屯象OS × 徐记海鲜 核心页面原型结构图 + 页面交互说明 V1》**





这版不是画高保真视觉，而是先把**最关键页面的结构、信息优先级、操作流、异常流、权限点**定下来。因为徐记海鲜样板阶段，最重要的不是“页面漂不漂亮”，而是这套系统能不能把四件事真正跑通：



1. 交易不断单
2. 出餐不失控
3. 折扣损耗可控
4. 总部能穿透看到并推动整改





这和前面已经定下的产品骨架、IA、页面清单是一致的：优先打透交易中台、商品供应链中台、日清日结操作层，再让经营分析和 Agent 形成闭环。  



我下面按**P0 核心页面**来出，因为这批页面最直接决定徐记样板能不能成立。

# **一、原型设计总原则**







## **1. 先任务，再信息**





门店一线页面不能先摆一堆图表，而要先让人完成动作：



- 迎宾先分台
- 服务员先点单
- 后厨先出餐
- 收银先结账
- 店长先处理异常







## **2. 一屏只服务一个主目标**





例如：



- 桌台总览页的主目标是“找台、看台、开台”
- 订单页的主目标是“点菜、改菜、催菜、结账前控制”
- KDS 页的主目标是“接单、排序、出餐、异常上报”







## **3. 异常永远前置**





凡是影响营业连续性的东西，都不放到深层菜单里：



- 折扣超权限
- 出餐超时
- 缺料卡单
- 挂账超额度
- 长短款异常







## **4. 门店端重状态，总部端重穿透**





门店页面要突出“现在发生了什么”；

总部页面要突出“哪里有问题、谁在负责、结果如何”。

## **1）门店工作台** 

## **/pos/dashboard**







### **页面定位**





POS 默认首页。

给迎宾、服务员、收银、店长一个“当下营业状态总入口”。





### **原型结构图**



```
┌──────────────── 顶部栏 ────────────────┐
│ 门店名 | 班次 | 当前时间 | 网络状态 | 同步状态 | 用户菜单 │
├──────────────────────────────────────┤
│ 今日营业快照                          │
│ 营收 | 在店桌数 | 待到店预订 | 待结账单 | 出餐超时 │
├───────────────┬──────────────────────┤
│ 左侧：我的待办 │ 右侧：实时异常提醒     │
│ - 待接待预订   │ - 折扣超权限           │
│ - 待开台桌台   │ - 出餐超时             │
│ - 待催菜订单   │ - 缺料断货             │
│ - 待结账订单   │ - 收银异常             │
├───────────────┴──────────────────────┤
│ 中部快捷入口                          │
│ [预订台账] [桌台总览] [开台点单] [收银结账] [交班] │
├──────────────────────────────────────┤
│ 底部：今日趋势条 / 最近操作 / 系统通知             │
└──────────────────────────────────────┘
```



### **交互说明**





进入页面后按角色显示不同内容：



- 迎宾：预订、排队、入座优先
- 服务员：桌台、催菜、客诉优先
- 收银：待结账、退款、交班优先
- 店长：异常、巡航、复盘优先







### **关键交互**





- 点击“待到店预订”跳转预订台账并自动筛选“即将到店”
- 点击“出餐超时”跳转 KDS 异常详情或桌台订单详情
- 点击“折扣超权限”打开审批抽屉
- 网络异常时顶部显示黄条；离线时显示红条，并展示“可点/可结/待同步”状态







### **页面目标**





让门店所有角色进系统第一眼就知道：

**今天营业怎么样、我现在先处理什么、哪里有风险。**

## **2）预订与排队台账** 

## **/pos/reservations/list**







### **页面定位**





迎宾主工作台。

处理散客预订、包厢预订、宴席预订、排队、签到、入座。





### **原型结构图**



```
┌──────────── 顶部筛选栏 ────────────┐
│ 日期 | 午市/晚市 | 类型 | 状态 | 搜索(姓名/电话/企业) │
├───────────────────────────────────┤
│ 左：今日预订列表                    │ 右：候位与到店面板  │
│-----------------------------------│--------------------│
│ 时间 | 客户 | 人数 | 类型 | 状态   │ 当前排队人数        │
│ 17:30 王总 8人 包厢 已确认         │ 已到店/待入座       │
│ 18:00 李女士 4人 散台 即将到店     │ 过号/催叫           │
│ ...                               │                    │
├───────────────────────────────────┤
│ 底部操作栏：新建预订 | 到店签到 | 分配桌台 | 改期 | 取消 │
└───────────────────────────────────┘
```



### **交互说明**







#### **预订主流程**



新建预订 → 选择客户/新客 → 选择人数/场景 → 选择散台或包厢 → 录入时间与备注 → 提交





#### **到店流程**



点击“到店签到”后：



- 若有合适桌台，直接进入“分配桌台”
- 若无桌台，进入“候位状态”，自动加入排队队列







#### **包厢流程**



包厢预订时联动展示：



- 包厢容量
- 低消
- 已预留情况
- 接待等级标签







### **关键交互**





- 同客户历史预订自动联想
- 高价值客户显示特殊标识
- 企业客户显示“协议单位”标签
- 临近到店 30 分钟自动高亮
- 过号后自动进入“待跟进”







### **异常流**





- 桌台不足：弹出推荐替代桌台/包厢
- 包厢冲突：展示冲突原因与可替代方案
- 企业挂账客户无授权：禁止标记为企业宴请

## **3）桌台/包厢总览** 

## **/pos/floor/map**







### **页面定位**





这是徐记海鲜样板里最关键的“正餐经营引擎”页面。

不是简单桌台列表，而是门店实时席位调度中心。





### **原型结构图**



```
┌──────── 顶部控制栏 ────────┐
│ 区域切换 | 楼层 | 散台/包厢 | 状态筛选 | 搜索桌号 │
├───────────────────────────┤
│ 左侧：桌台地图主画布                                │
│ [A01 空台] [A02 已预订] [A03 用餐中 48min] [VIP1 包厢] │
│ [B01 待清台] [B02 锁台] [B03 用餐中 待上甜品]         │
├───────────────┬───────────┤
│ 右侧：桌台详情抽屉         │ 快捷动作栏              │
│ 桌号/包厢名                │ [开台] [转台] [并台]    │
│ 当前状态                   │ [清台] [锁台] [备注]    │
│ 当前订单/人数/服务员        │                         │
│ 用餐时长/催菜次数/客诉      │                         │
└───────────────────────────┘
```



### **桌台状态规范**





建议固定 8 种颜色/状态语义：



- 空台
- 已预留
- 待入座
- 用餐中
- 待结账
- 待清台
- 锁台
- 维修/停用







### **交互说明**







#### **开台**



点击空台 → 打开右侧详情 → 点击开台 → 选择人数/服务员/是否关联预订 → 进入点单页





#### **转台/并台/拆台**



点击用餐中桌台 → 选择目标台 → 系统展示：



- 菜品是否随台转移
- 是否保留原服务员
- 是否影响账单拆分







#### **包厢调度**



包厢显示低消、接待等级、预订来源、当前进度。

适合商务宴请场景。





### **关键交互**





- 用餐中桌台显示计时器
- 即将超时桌台高亮
- 出餐异常桌台出现红点
- 待结账桌台显示金额角标
- 桌台点击不跳页面，先右侧抽屉，减少操作中断







### **页面目标**





让门店现场一眼看到：

**哪有空台、哪桌快超时、哪桌能翻台、哪桌有异常。**

## **4）开台点单页** 

## **/pos/orders/:orderId**







### **页面定位**





全系统最核心页面。

这里承接：点菜、加菜、退菜、称重菜、赠菜、折扣、催菜、拆单。





### **原型结构图**



```
┌──────── 顶部订单头 ────────┐
│ 桌号/包厢 | 人数 | 服务员 | 开台时长 | 会员/客户标签 │
├──────────────┬────────────┤
│ 左侧：菜品分类区 │ 中间：菜品列表区          │
│ 冷菜/热菜/海鲜   │ 菜品卡：名称/价格/库存/标签 │
│ 酒水/主食/套餐   │ 支持搜索/推荐/最近点过      │
├──────────────┴────────────┤
│ 右侧：购物车/订单区                            │
│ 菜品明细                                       │
│ 数量 / 口味 / 做法 / 备注 / 称重 / 小计         │
│-----------------------------------------------│
│ 底部操作：发送后厨 | 加菜 | 退菜 | 赠菜 | 折扣 | 结账 │
└───────────────────────────────────────────────┘
```



### **交互说明**







#### **点菜流程**



选择分类 → 点选菜品 → 如需口味/做法则弹规格层 → 加入右侧订单区 → 点击“发送后厨”





#### **时价称重菜**



点击海鲜类菜品后进入称重弹层：



- 重量

- 单价

- 加工方式

- 理论毛利提示

  确认后写入订单







#### **加菜**



已下单订单继续加菜时：



- 新增菜品带“追加”标识
- 若档口拥堵，展示预计出餐时间







#### **退菜**



退菜必须选原因：



- 点错

- 客诉

- 缺料

- 重做

- 其他

  并按权限判断是否需要审批







#### **折扣**



点击“折扣”打开折扣抽屉：



- 整单折扣

- 单品折扣

- 券/权益抵扣

- 企业协议价

  系统同时显示：

- 折扣前金额

- 折扣后金额

- 毛利底线提醒

- 是否超权限







### **关键交互**





- 菜品卡显示“沽清”“推荐”“招牌”“高毛利”等标签
- 订单右侧始终固定，避免切换丢失上下文
- 已发送菜品与未发送菜品视觉分层
- 若某菜缺料，系统推荐替代菜
- 若同桌客户有历史偏好，可顶部展示偏好提示







### **异常流**





- 毛利跌破底线：禁止直接提交折扣，需审批
- 菜品沽清：禁止下单，推荐替代菜
- 网络异常：本地缓存待同步，并提示“已本地保存”
- 退菜已出餐且已上桌：必须店长审批







### **页面目标**





保证一线在高峰期也能快速完成：

**点菜快、改菜稳、折扣可控、称重不出错。**

## **5）收银结账台** 

## **/pos/cashier/:orderId**







### **页面定位**





从“待结账单”进入，完成付款、核销、挂账、退款、打印。





### **原型结构图**



```
┌──────── 订单结账头部 ────────┐
│ 桌号 | 客户 | 开台时长 | 服务员 | 应收金额 │
├────────────────────────────┤
│ 左侧：账单明细                │ 右侧：支付与抵扣区   │
│ 菜品/数量/小计                │ 支付方式             │
│ 服务费/包厢费                 │ 现金/扫码/银行卡     │
│ 折扣/券/抹零                  │ 储值/权益/挂账       │
│ 实收汇总                      │                      │
├────────────────────────────┤
│ 底部操作：试算 | 核销券 | 企业挂账 | 确认收款 | 打印小票 │
└────────────────────────────┘
```



### **交互说明**







#### **常规收款**



确认账单 → 选择支付方式 → 输入/回填金额 → 确认收款 → 打印





#### **混合支付**



支持多段支付：



- 现金 + 扫码
- 储值 + 现金
- 券 + 企业挂账补差







#### **企业挂账**



选择协议单位 → 校验额度 → 选择签单人/授权人 → 提交挂账





#### **券核销**



打开券面板后系统自动校验：



- 使用门槛
- 适用品类
- 冲突权益
- 时间限制







### **关键交互**





- 左右双栏，账单和支付不切屏
- 券核销后账单实时重算
- 企业挂账显示剩余额度
- 折扣来源、审批人、核销记录可追溯







### **异常流**





- 挂账超额度：禁止结账，需审批
- 券冲突：提示不可叠加原因
- 反结账：仅店长以上可发起
- 长短款疑似异常：自动挂起到交班稽核







### **页面目标**





做到：

**结账快、挂账准、核销清、事后可审计。**

## **7）KDS 出餐总看板** 

## **/kds/board**







### **页面定位**





后厨总调度页面。

目标是把“更快出餐与翻台”做成系统能力。





### **原型结构图**



```
┌──────── 顶部栏 ────────┐
│ 档口切换 | 午市/晚市 | 总任务数 | 超时数 | 缺料数 │
├───────────────────────┤
│ 左：待制作队列          │ 中：制作中队列          │ 右：异常队列 │
│ 票据卡片               │ 票据卡片               │ 超时/重做/卡单 │
│ 桌号/菜品/下单时间      │ 已开始时间/预计剩余      │ 原因/优先级   │
├───────────────────────┤
│ 底部：快捷动作 [开始] [完成] [加急] [缺料] [重做] │
└───────────────────────┘
```



### **交互说明**







#### **接单流程**



待制作 → 开始制作 → 制作中 → 出餐完成





#### **缺料流程**



点击“缺料” → 选择原因 → 回传前厅和桌台 → 触发替代建议或退菜流程





#### **加急流程**



前厅催菜后，票据卡片高亮并置顶





### **关键交互**





- 卡片显示桌号、菜品、数量、口味、等待时长
- 超时卡片自动进入异常列
- 档口可按负载动态排序
- 重做单显示明显标签，避免混淆







### **页面目标**





后厨一眼知道：

**先做什么、哪单快超时、哪单缺料、哪里堵了。**

## **8）日清日结巡航页** 

## **/crew/daily/cruise**







### **页面定位**





店长/值班经理的营业中巡航页面。

把“门店执行”从经验式管理变成节点式管理。





### **原型结构图**



```
┌──────── 巡航头部 ────────┐
│ 门店 | 班次 | 当前阶段 | 完成率 | 异常数 │
├─────────────────────────┤
│ 左侧：今日流程节点           │ 右侧：节点详情/填报区   │
│ 开店准备 ✓                  │ 当前节点说明            │
│ 营业巡航 진행中             │ 检查项列表              │
│ 高峰值守 待开始             │ 图片/备注/责任人        │
│ 闭店盘点 待开始             │ 提交/转异常             │
├─────────────────────────┤
│ 底部：异常汇总 / 指派整改 / 历史巡航记录          │
└─────────────────────────┘
```



### **交互说明**





- 每个节点可填写检查结果、备注、图片
- 发现异常可一键转任务
- 高峰时段自动弹出“高峰值守”节点
- 未完成节点在时间到达后高亮提醒







### **关键交互**





- 节点必须可签字/确认
- 异常支持指派给责任人
- 同一问题可关联到店长复盘







### **页面目标**





让“开店—营业—闭店”成为真正的系统化操作层。

这和你之前明确推进的门店全天业务流程节点管理、日清日结目标完全一致。

# **四、关键弹层/抽屉原型**





除了主页面，徐记样板还有几个高频关键弹层必须先定义。

- 处理结果
- 是否复访





目标：

把客诉从口头处理变成可复盘数据。

字段：



- 异常类型
- 责任人
- 完成时限
- 整改要求
- 是否升级区域





目标：

让预警能落到人。

# **五、状态机与交互约束建议**





为了后面开发不跑偏，建议先把这几组状态写死。





## **1. 桌台状态机**





空台 → 已预留 → 待入座 → 用餐中 → 待结账 → 待清台 → 空台



异常分支：



- 锁台
- 维修
- 临时停用







## **2. 订单状态机**





草稿 → 已下单 → 制作中 → 部分上菜 → 已上齐 → 待结账 → 已结账



异常分支：



- 部分退菜
- 整单取消
- 挂账中
- 退款中







## **3. 巡航节点状态机**





未开始 → 进行中 → 已完成 → 异常待处理 → 已闭环





## **4. 整改任务状态机**





新建 → 已指派 → 处理中 → 待复查 → 已关闭
