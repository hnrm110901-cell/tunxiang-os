### §1.2 PaymentSaga S1→S3 跨步骤占位锁机制（issue #537）

**问题**：FOR UPDATE 在 transaction commit 释放，无法跨 saga 步骤；双 worker 灾难恢复放大风险。

**候选**：
- A. 条件 UPDATE 占位（S1 直接 UPDATE status='paying'）— 与现有 _complete_order 风格一致
- B. saga_locks 专表 + heartbeat — 跨步骤 + 天然支持 SKIP LOCKED 灾难恢复
- C. PG advisory lock — 与 connection pool 模型不兼容（saga S2 释放连接是模式）

**建议决议**：A 走 architect agent 评估（含 transition_order state machine 守卫影响 + 'paying' 中间态 emit_event 时序 + recover_pending_sagas SKIP LOCKED 复用），W22 守门会前出评估结论。
