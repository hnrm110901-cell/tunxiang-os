## 三、数据质量综合评分

通过 `GET /api/v1/analytics/data-quality/{merchant_code}` 获取自动评分。

### 评分权重汇总

| 维度 | 权重 | 最低通过分 | 说明 |
|------|------|-----------|------|
| 门店数据完整率 | 20% | 100 | 门店信息必填完整 |
| 菜品数据完整率 | 20% | 80 | 数量和字段完整性 |
| 会员数据完整率 | 15% | 100 | ≥ 5 位即可满分 |
| 历史订单数据 | 15% | 100 | ≥ 20 条即可满分 |
| KPI 权重已配置 | 10% | 100 | 有配置即可满分 |
| 桌台数据完整率 | 10% | 100 | ≥ 5 张即可满分 |
| 主键一致性 | 10% | 100 | 0 孤立订单 |
| **总分** | **100%** | **≥ 70** | **闸门通过线** |

### 等级划分

| 分数区间 | 等级 | 含义 |
|----------|------|------|
| ≥ 95 | A+ | 优秀，可上线 |
| ≥ 90 | A | 良好，可上线 |
| ≥ 85 | B+ | 达标，推荐上线 |
| ≥ 80 | B | 基本达标 |
| ≥ 70 | C+ | 最低通过线 |
| ≥ 60 | C | 不达标，需补充数据 |
| < 60 | D | 不可上线 |

## 四、数据变更流程

### 4.1 新增商户

```bash
# 1. 创建种子数据脚本
cp scripts/seed_czyz.py scripts/seed_{new_code}.py

# 2. 注册数据质量检查
#    修改 services/tx-analytics/src/api/merchant_data_quality_routes.py
#    在 _DEMO_TENANTS 中添加新商户映射

# 3. 注册发布闸门检查
#    修改 scripts/release-gate.sh 的 MERCHANTS 列表

# 4. 更新本清单
```

### 4.2 商户数据更新

- 直接运行对应 seed 脚本（幂等，`ON CONFLICT` upsert）
- 运行后重新获取数据质量评分验证完整性

### 4.3 数据重置

```bash
# 清空指定商户数据（先删除后重新 seed）
python3 scripts/seed_{merchant}.py  # 重新执行
```
