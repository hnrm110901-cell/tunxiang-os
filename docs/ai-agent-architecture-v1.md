# 屯象OS 原生 AI Agent 分层架构 + 业务流适配矩阵 V1

> 本文档定义屯象OS AI Agent 系统的八层架构设计和业务流适配策略。

---

## 1. 方案定位

交易操作系统底座 + 原生 AI Agent 协同层 + 中国多硬件门店运行时

- 底层交易流：代码和状态机主导
- 跨系统协同流：Agent + 工具调用主导
- 经营分析与动作建议：原生 Agent 主导
- 门店终端运行：Android First，iPad 补充

---

## 2. 设计原则

1. **先系统，后智能** — 先打通业务事件，再挂 Agent
2. **Agent 不替代交易内核** — Agent 可理解、推荐、解释、提醒、编排；不能直接替代下单、支付、退款、对账
3. **原生 AI 不是问答框，而是任务入口** — 用户来完成任务，不是来聊天的
4. **门店必须断网可营业** — 必须有 Store Edge Runtime

---

## 3. 八层架构

```
L0  终端交互层      收银POS / 迎宾预订 / 服务员手持 / 厨房KDS / 顾客扫码 / 店长端 / 总部端
L1  场景会话层      意图识别 + 角色识别 + 上下文注入 + 多轮任务拆解
L2  Agent 编排层    总控 Agent + 8个专业 Agent
L3  Tool/MCP 网关层  受控工具调用（预订/等位/桌台/订单/厨房/支付/会员/报表等）
L4  交易与业务域服务  Reservation / Waitlist / Table / Order / Kitchen / Payment / Member / Report
L5  状态机与规则引擎  桌态/订单/出品/结算状态机 + 会员/营销/审批/风控规则
L6  数据与智能底座   事件总线 + ODS/DWD/DWS/ADS + 实时指标 + Agent Memory
L7  基础设施与设备   云端 + Edge Runtime + Android POS + iPad + KDS + 外设 + 支付接口
```

---

## 4. Agent 角色分工

### 总控 Agent (Dispatcher)
- 理解用户目标 → 判断可执行性 → 选择专业 Agent → 控制串行/并行 → 汇总结果

### 8 个专业 Agent

| # | Agent | 职责 | 优先级 |
|---|-------|------|--------|
| 1 | 迎宾预订 Agent | 预订确认、改期改桌、顾客偏好、包厢推荐 | P0 |
| 2 | 等位桌台 Agent | 等位排序、桌台调度、翻台预估、VIP优先 | P0 |
| 3 | 点单服务 Agent | 菜品推荐、套餐搭配、忌口提醒、加菜建议 | P1 |
| 4 | 厨房协同 Agent | 拥塞识别、出菜超时、催菜判断、停售建议 | P1 |
| 5 | 会员增长 Agent | 会员识别、券推荐、沉睡唤醒、复购策略 | P1 |
| 6 | 结账风控 Agent | 结账异常、折扣解释、退款风险、发票异常 | P1 |
| 7 | 店长经营 Agent | 经营播报、午晚市复盘、人效分析、整改任务 | P0 |
| 8 | 总部分析 Agent | 多门店对比、经营洞察、预警聚合、周报草拟 | P0 |

## 5. 业务流适配矩阵

适配模式：
- **A**: 原生 Agent 主导
- **B**: Agent + 状态机协同
- **C**: 代码/状态机主导，Agent 仅辅助

| 业务流 | 适配模式 | Agent 价值 | 系统控制部分 |
|--------|---------|-----------|-------------|
| 预订 | B | 意图识别、桌位推荐、确认话术 | 预订锁台、时段容量、超时释放 |
| 等位 | B | 等待预估、优先级建议、流失预警 | 队列顺序、叫号规则、到号超时 |
| 桌台管理 | C | 翻台预测、调度建议 | 桌态状态机、冲突校验 |
| 点单 | C | 推荐搭配、忌口提示 | 下单、改单、权限、价格 |
| 厨房出品 | C | 拥塞识别、优先级建议 | 分单路由、工位状态、出菜确认 |
| 会员识别 | B | 自动识别、权益推荐 | 会员主档、积分、券核销 |
| 结账支付 | C | 风险提醒、异常解释 | 支付、退款、发票、审计 |
| 营销触达 | A | 人群细分、文案生成、触达编排 | 活动规则、发券额度 |
| 报表分析 | A | 自动总结、经营问答、归因 | 指标口径、数据权限 |
| 日清日结 | B | 自动复盘、整改项生成 | 班结、交班、对账、锁账 |
| 巡店整改 | A | 任务生成、执行跟踪 | 审批流、责任归属 |
| 总部预警 | A | 异常识别、对策生成 | 指标计算、权限 |

## 7. 原生 AI 入口类型

1. **任务输入入口** — "安排今晚8点包厢"、"帮我看今天午市问题"
2. **异常卡片入口** — 系统主动推送：等位流失、出菜超时、高额折扣
3. **建议执行入口** — 一键发送确认、一键生成整改、一键发券
4. **复盘入口** — 午市复盘、晚市复盘、周经营总结
