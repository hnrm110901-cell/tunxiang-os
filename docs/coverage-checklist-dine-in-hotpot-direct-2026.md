# 正餐 + 火锅连锁直营 — 屯象OS 必须覆盖功能与报表明细

> **目标客户**：中式正餐、火锅为主，**直营**门店网络（可含多区域/多品牌，但**不以加盟费/加盟商对账为签约主路径**）。  
> **用途**：产品验收、研发排期、商务承诺边界；与 [天财报表对照全量](gap-tiancailong-v2-checklist-hq-reports.md) 配合使用——本文是**子集 + 屯象表述**，非天财全文复制。  
> **日期**：2026-04

---

## 一、必须覆盖的功能明细（按域）

说明：**P0** = 不上线难以替换竞品、影响签单/日结；**P1** = 强期待、可分期但须在首期后 6 个月内闭合。

### 1. 门店交易（`web-pos` / `tx-trade`）

| 编号 | 能力 | 优先级 | 正餐要点 | 火锅要点 |
|------|------|--------|----------|----------|
| F1 | 开台 / 客位 / 区域 / 并台换台 / 锁台 | P0 | 包厢、大厅分区 | 大桌、拼桌、计时场景预留 |
| F2 | 点菜 / 加单 / 规格 / 做法 / 备注 / 半份 | P0 | 宴请、时价 | **锅底/蘸料/饮料**与涮品分链路 |
| F3 | **套餐 / 组合**（含可选组、可换菜） | P0 | 宴会套餐 | **套餐锅**、荤菜组合、自助小料台计费规则 |
| F4 | 称重 / 时价品 | P0 | 海鲜活鲜 | 牛羊肉按斤、现切类 |
| F5 | 等叫 / 叫起 / 催菜 / 划菜协同 KDS | P0 | 出餐节奏 | 锅底先上、爆单时序 |
| F6 | 折扣 / 赠菜 / 改价 **权限 + 留痕**（Agent 可叠加稽核） | P0 | 签批、免单 | 高频敏感，需与折扣守护一致 |
| F7 | 多种支付、组合支付、找零、开钱箱 | P0 | — | — |
| F8 | 预结 / 分单打印 / 发票 / 小票模板 | P0 | 宴请分席 | 加汤加火小票 |

| F11 | **最低消费、服务费、茶位/餐位费**（可配置） | P0 | 包厢常见 | 蘸料费按位、锅底费 |
| F12 | **存酒**（寄存、支取、结余、会员/桌关联） | P1 | 中高端正餐 | 啤酒/白酒寄存 |

| F14 | **团购券 / 平台券核销** | P0 | 引流 | 火锅极高频 |
| F15 | 挂账 / 签单（直营下 **经理签单、企业挂账**） | P1 | 协议客户 | 大客户少但要有出口 |
| F16 | 外卖平台接单（厨房分流、渠道标识） | P1 | 增量收入 | 火锅外卖占比因品牌而异，**报表要对账** |

## 五、直营场景下可降优先级的天财模块（避免范围膨胀）

- **加盟商管理、加盟费、特许经营报表** → P2 或单独立项。  
- **智慧商街、多商户合一收银** → 非本文客群主路径。  
- 带 **品牌缩写后缀** 的增值报表（如 XJD、DYHG 定制版）→ 用 **通用 P0 表 + 参数化门店/品牌** 替代，不逐客户抄表。
