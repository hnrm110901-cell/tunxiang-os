等保三级合规要求：OWASP Top 10 防护、CC 攻击防护、SQL 注入防护、操作日志留存。

- 服务器：腾讯云广州 42.194.229.21
- 防护域名：*.tunxiangos.com
- 目标版本：腾讯云 Web 应用防火墙（入门版）

## 二、添加防护域名

### 2.1 进入 WAF 控制台

腾讯云控制台 → 安全 → Web 应用防火墙 → 域名接入 → 添加域名。

### 2.2 填写域名配置

| 字段 | 值 |
|------|----|
| 域名 | api.tunxiangos.com |
| 协议 | HTTPS（443） |
| 回源协议 | HTTP |
| 回源 IP | 42.194.229.21 |
| 回源端口 | 8000 |
| 是否开启 HTTPS | 是，上传证书 |

**重复以上步骤**，依次添加所有子域名：
`hub.tunxiangos.com` / `os.tunxiangos.com` / `pos.tunxiangos.com` / `api.tunxiangos.com`

### 2.3 获取 CNAME 接入地址

添加完成后，WAF 会分配一个 CNAME 地址，形如：
`xxxx.saas.waf.qcloud.com`

## 三、修改 DNS（流量切换到 WAF）

登录域名 DNS 控制台（腾讯云 DNS 解析 / DNSPod）：

```
# 删除原来的 A 记录
api.tunxiangos.com  A    42.194.229.21

# 添加 CNAME 记录（指向 WAF 接入地址）
api.tunxiangos.com  CNAME  xxxx.saas.waf.qcloud.com
```

DNS 生效后，所有流量经过 WAF 清洗再回源到服务器。

**验证 DNS 生效**：
```bash
nslookup api.tunxiangos.com
# 应解析到 WAF IP（非 42.194.229.21）
```

**验证 WAF 正常代理**：
```bash
curl -I https://api.tunxiangos.com/health
# 响应头中应看到 WAF 注入的头：X-WAF-RULE 或类似字段
```

## 四、开启防护规则

### 4.1 Web 基础防护（OWASP Top 10）

路径：WAF 控制台 → 防护配置 → Web 安全 → 基础防护。

| 规则集 | 模式 | 说明 |
|--------|------|------|
| SQL 注入防护 | 拦截 | 防止数据库被非法查询 |
| XSS 跨站脚本 | 拦截 | 防止恶意脚本注入 |
| 命令注入 | 拦截 | 防止 OS 命令执行 |
| 文件包含 | 拦截 | 防止路径遍历攻击 |
| Webshell 检测 | 拦截 | 防止后门文件上传 |
| 扫描器识别 | 观察（初期） | 先观察1周，确认无误报后改拦截 |

**建议**：新接入第一周先开启"观察"模式，确认无误报后切换为"拦截"模式。

### 4.2 CC 防护（DDoS 应用层防护）

路径：WAF 控制台 → 防护配置 → CC 防护。

| 接口 | 频率限制 | 匹配维度 | 动作 |
|------|---------|---------|------|
| /api/v1/auth/* | 10次/分钟/IP | 源 IP | 验证码（优于直接封禁） |
| /api/* | 200次/分钟/IP | 源 IP | 人机验证 |
| 全局 | 500次/分钟/IP | 源 IP | 封禁 10 分钟 |

### 4.3 自定义规则：Admin 接口 IP 白名单

路径：WAF 控制台 → 防护配置 → 自定义规则 → 新建规则。

**规则：保护 Admin 接口**

| 字段 | 值 |
|------|----|
| 规则名称 | admin-ip-whitelist |
| 匹配条件 | URL 包含 `/api/v1/admin/` AND 源IP 不在白名单 |
| 动作 | 拦截（403） |
| 优先级 | 1（最高） |

白名单 IP 填写：
- 李淳 Tailscale IP（`100.x.x.x`，通过 `tailscale ip -4` 获取）
- 办公室出口 IP（可选）

**规则：拦截常见扫描路径**

| 字段 | 值 |
|------|----|
| 规则名称 | block-scanner-paths |
| 匹配条件 | URL 匹配正则 `(\.php|phpMyAdmin|wp-admin|\.env|\.git)` |
| 动作 | 拦截（404） |

## 五、日志接入 CLS（保留 180 天）

路径：WAF 控制台 → 日志服务 → 访问日志 → 开启日志。

| 配置项 | 值 |
|-------|----|
| 地域 | 广州（与服务器同地域，节省流量费） |
| 日志集 | txos-security（与 CLS 方案统一，见 cls-log-setup.md） |
| 日志主题 | waf-access-log |
| 保留时长 | **180 天**（等保三级要求） |
| 日志格式 | JSON |

## 六、告警配置

路径：WAF 控制台 → 告警设置（或通过腾讯云 Monitor 配置）。

### 6.1 短信告警规则

| 告警名称 | 触发条件 | 通知方式 |
|----------|---------|---------|
| 攻击激增 | 1分钟内拦截次数 > 100 | 短信 + 企微 |
| CC 攻击 | 触发 CC 防护 | 短信 |
| SQL 注入尝试 | 检测到 SQL 注入 > 10次/5分钟 | 短信 |
| 高危漏洞利用 | 检测到 Webshell / 命令注入 | 立即短信 |

### 6.2 配置告警接收人

腾讯云控制台 → 监控告警 → 通知模板：
- 接收人：李淳（手机号/邮箱）
- 通知渠道：短信 + 微信公众号推送

### 服务器安全组配置（防绕过 WAF）

腾讯云控制台 → 安全组 → 修改入站规则：

```
# 删除：
0.0.0.0/0  TCP  80   允许
0.0.0.0/0  TCP  443  允许

# 添加（WAF 回源 IP 段，从腾讯云文档获取最新列表）：
WAF回源IP段  TCP  80,443  允许
100.64.0.0/10  TCP  22  允许  # Tailscale
127.0.0.1/32   TCP  ALL 允许  # 本机
```
