# 演示话术 03 — 财务采购视角（CFO + 采购总监）

> 受众：徐记海鲜 CFO + 采购总监 + 内审
> 时长：40 分钟
> 核心诉求："钱能看清楚 + 采购能管控 + 税务合规"

## 开场（2 分钟）

> "赵总，23 套系统最大的痛是月结——现金账、POS 流水、会员充值、外卖佣金、
> 活鲜损耗，跨 6 个系统手工对账。今天演示屯象如何一步打通。"

## 场景 1：实时财务驾驶舱（10 分钟）

### 1.1 3 家门店 P&L 实时透视

- 访问 `web-admin → 财务 → P&L Dashboard`
- 演示：切换门店/品牌/时间维度，毛利 / 人工 / 租金 / 能耗实时刷新
- **亮点**：物化视图（v148 `mv_store_pnl`）秒级聚合，不拖累业务库

### 1.2 成本根因分析（Sprint D4a）

- 场景：长沙店 4 月食材成本从 38% 升到 45%
- 点击"根因分析" → Sonnet 4.7 给出：
  ```
  1. 3 月 20 日海鲜批发市场涨价 12%（高置信度）
  2. 鲜货库存损耗从 3% 升至 5%（管理问题）
  3. 菜品结构：高利海鲜菜品点击率下降 8%（需要营销）
  ```
- **亮点**：Prompt Cache 共享行业基准，月度成本降 ¥8k

### 1.3 预算预测（Sprint D4c）

- 演示：5 月预算自动生成 → CFO 审核 → approve/revise/escalate 三态
- **亮点**：基于历史 12 月 P&L × 季节性调整 × 行业 benchmark

## 场景 2：供应链管控（12 分钟）

### 2.1 活鲜追溯（Sprint B 合规）

- 演示：扫码 → 查看鲜虾来源（供应商 / 采购日 / 检疫证）
- **亮点**：一物一码，食安追溯 48h 内完成

### 2.2 食材效期预警

- 演示：库存预警 Agent → 冰柜温度异常 / 效期 < 48h 自动告警到店长手机
- **亮点**：边缘 Core ML 实时监测（Sprint D1 SOP 时间轴）

### 2.3 采购订单 + 供应商对账

- 演示：月度采购单自动生成（基于销量预测）
- 演示：供应商发票 → OCR → 自动匹配采购单 → 差异 > 3% 弹预警
- **亮点**：省采购对账时间 80%

## 场景 3：税务合规（8 分钟）

### 3.1 金税四期全电发票

- 演示：顾客索要发票 → 扫码填抬头 → 2 秒出票
- **亮点**：符合金税四期格式，不被退票（Tier 1 验证）

### 3.2 销项汇总 + 申报

- 演示：月末自动生成增值税销项汇总 → 一键导出税务申报文件
- **亮点**：与航天信息 / 诺诺网 / 百望云打通

### 3.3 异常交易审计

- 演示：财务稽核 Agent 自动标记：
  - 夜班退单率 > 5%
  - 会员折扣叠加超限
  - 礼品卡兑付超当日销售 30%
- **亮点**：决策留痕，便于审计

## 场景 4：资金流全景（8 分钟）

### 4.1 现金 / 支付宝 / 微信 / 会员储值 全渠道

- 演示：资金流 Sankey 图 → 每天各渠道占比 + 结算到账情况
- **亮点**：事件总线（v147）保真

### 4.2 会员储值流水 + 对账

- 演示：顾客充值 500 送 50 → 储值余额 + 赠送余额分账
- 演示：月度储值退费 → 自动生成红字发票
- **亮点**：CLV / 储值负债准确计量

### 4.3 平台佣金 + 补贴对账

- 演示：美团 4 月佣金单 → 自动与 canonical_delivery_orders 对账
- 演示：差异 > ¥100 自动升级到 dispute 工作流（E4）

## Q&A 准备

### Q1: "我们现在手工做月结，你们的对账准确率多少？"
A: DEMO 数据 99.7% 匹配；剩余 0.3% 由 dispute 工作流（E4）处理，可追溯。

### Q2: "税务监管越来越严，你们跟得上吗？"
A: 金税四期已对接 3 个服务商（航天信息 / 诺诺网 / 百望云）；新政策 48h 内更新规则。

### Q3: "供应商不用你们系统，怎么对账？"
A: OCR + PDF 解析 + AI 匹配，匹配不上的转人工队列（运营总监审核）。

### Q4: "审计配合怎么做？"
A: 事件总线 append-only，任何数据都可追溯到事件源；审计人员按 tenant_id 只读访问 BI 系统。
