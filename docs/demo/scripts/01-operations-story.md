# 演示话术 01 — 运营故事线（徐记老板视角）

> 受众：徐记海鲜董事长 + 运营总监 + IT 总监
> 时长：45 分钟
> 核心诉求："屯象OS 怎么替代现有 23 套系统"

## 开场（3 分钟）

> "徐总，您现在门店跑 23 套系统——收银、KDS、库存、排班、外卖、财务、会员、巡店…
> 每套系统厂商不同，数据不通。月结对账要 3 天，活动复盘要 1 周。
> 今天给您看的不是第 24 套系统，而是**一套系统替代全部**。
>
> 我会先跑通一家门店一天的完整链路，再展示总部视角的决策层。"

## 场景 1：早班开店（8 分钟）

### 1.1 日清日结 + 现金保险箱交接

- 访问 `web-admin → 运营流程 → 日清日结`
- 演示：夜班交接 → 现金账核对 → 食材损耗录入 → 明日采购单生成
- **亮点**：系统自动比对 POS 流水 + 现金保险箱差额，差额 >50 元即预警

### 1.2 SOP 任务推送到员工手机

- 演示：店长手机收到 8:00 "开店检查清单"（冰箱温度 / 消毒记录 / 食材效期）
- **亮点**：SOP 时间轴引擎（Sprint D1），每店每日 30+ 自动任务

### 1.3 后厨 KDS 上线

- 演示：后厨平板 → 订单队列按档口分流
- **亮点**：出餐时间预测（Core ML 边缘推理，Sprint D1）

## 场景 2：午间高峰（10 分钟）

### 2.1 POS 收银：扫码 + 会员 + 折扣

- 演示：服务员手机 → 扫桌二维码 → 客人自助点餐 → 确认下单
- 收银台一键买单 → 识别 VIP 会员 → 折扣叠加 → 支付宝扫码
- **亮点**：Tier 1 路径全绿（P99 < 200ms）

### 2.2 折扣守护 Agent

- 演示：服务员输入 "8 折" → Agent 立即拦截："毛利低于 15% 红线，仅允许 85 折"
- **亮点**：三条硬约束（毛利 / 食安 / 客户体验），决策留痕

### 2.3 外卖订单进线

- 演示：美团 webhook 推送 → canonical schema 自动归一化（Sprint E1）
- **亮点**：一条订单同步 KDS + 财务 + 库存扣减，不再手动录入

## 场景 3：下午巡店 + 数据复盘（8 分钟）

### 3.1 巡店 Agent 自动质检

- 演示：摄像头 → 识别"未戴帽子 / 地面油污 / 垃圾桶溢出"
- **亮点**：违规自动生成 SOP 纠正任务 + 升级通道

### 3.2 实时驾驶舱

- 演示：总部看板 → 3 家门店实时翻牌率 / 人效 / 毛利
- **亮点**：物化视图（v148）秒级刷新，不拖累业务库

### 3.3 Sonnet AI 成本根因分析

- 演示：上海店 4 月食材成本突增 15% → Sonnet 给出：`3 月 8 日海鲜批发市场涨价 12% + 库存管理损耗从 3% 升至 5%`
- **亮点**：Prompt Cache 命中率 ≥ 75%，月度成本降 ¥12k

## 场景 4：晚间闭店 + 异议处理（8 分钟）

### 4.1 外卖异议自动草稿

- 演示：美团推送"漏菜"投诉 → 系统匹配 `missing_item_full_refund` 模板 → 商家 1 次点击即可提交
- **亮点**：E4 异议工作流，15 个模板覆盖 11 类异议，历史结案率 95%+

### 4.2 薪资异常 + 预算预测

- 演示：上海店人工成本占比 32% → Sonnet 标 `compliance_breach legal_flag=true`
- 演示：5 月预算预测 → 净利率预测 17% ± 2pp
- **亮点**：D 批次 Prompt Cache 共享基准

### 4.3 日结 + 备用金

- 演示：收银 POS 生成 `daily_settlement` → 店长手机审核 → 备用金保险箱记录
- **亮点**：CRDT 冲突解析（断网 4h 无数据丢失）

**关键承诺**：
1. 所有现有数据迁移（90 天内）
2. 任何一个旧系统数据可以回导（防止供应商锁定）
3. 不改门店任何硬件（外设走商米 SDK）

## Q&A 准备（常见问题）

### Q1: "你们 AI 靠谱吗？Claude 会不会胡说？"
A: 我们用三条硬约束 + 决策留痕（每个 Agent 决策有 reasoning + context），误决策可以复盘 + 追溯 + 回滚。

### Q2: "断网怎么办？"
A: Mac mini 本地 DB + CRDT 同步，断网 4h 内收银不受影响（Sprint H 验证）。

### Q3: "数据安全？"
A: PostgreSQL RLS 租户隔离 + KMS 加密敏感字段 + 零信任网络（Tailscale）。
