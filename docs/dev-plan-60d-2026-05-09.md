## 6. 起手协议（每会话强制 5 步）

```bash
cd /Users/lichun/tunxiang-os
git fetch git@github.com:hnrm110901-cell/tunxiang-os.git main:refs/remotes/origin/main
git rev-parse origin/main HEAD                            # ff 是否需要
gh pr list --state open --limit 20                        # PR queue 当前
git worktree list                                         # 工作树状态
# 怀疑某文件有漂移：git show origin/main:<file> 比对真实 baseline
# 怀疑 worktree 有 WIP：diff <worktree-file> <(git show origin/main:<path>)
```

**强制规则**：
- worktree 隔离（每个 PR 独立 worktree，`.tunxiang-p0-worktrees/<task-id>/`）
- SSH 显式 push（`git push git@github.com:hnrm110901-cell/...`，不动 origin remote）
- Co-authored-by 占位（`你的名字 <你的邮箱@example.com>` + `Claude Opus 4.7 (1M context) <noreply@anthropic.com>`）
- TDD 必须（Tier 1 路径，红测试 → 实现 → 双 commit 留痕）
- 提交规范 `[type]([service]): 描述 [Tier级别]`
- 会话结束 DEVLOG.md + docs/progress.md 必须更新（CLAUDE.md §16/§18）
- `merge` / 推送 / 评论 PR 等共享状态变更 → user 确认后再动
