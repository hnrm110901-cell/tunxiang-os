## 三、分阶段开发路线图

### Phase 1: 系统可跑（第1-4周）— 解决上线阻塞

```
Week 1-2: 支付SDK + 财务路由接通
  ├─ P0-1: 对接收钱吧聚合支付SDK（微信/支付宝/银联）
  │        替换 payment_gateway.py 中 3 个 Mock 方法
  │        测试: 扫码支付 → 回调 → 订单状态更新
  ├─ P0-2: tx-finance 15个API路由接通Service层
  │        注入 AsyncSession，调用 store_pnl + finance_analytics
  │        测试: 日结报表 → P&L → 成本率
  └─ 验收标准: 能完成 "点餐→支付→日结" 闭环

Week 3-4: 同步引擎 + 加盟管理
  ├─ P0-3: sync-engine 4个核心方法实现
  │        PG logical replication → 增量变更检测
  │        ON CONFLICT UPSERT → 冲突解决
  │        水位表 → 断点续传
  │        测试: 本地PG ↔ 云端PG 双向同步
  ├─ P0-4: tx-franchise 服务新建
  │        加盟商注册 → 合同 → 供货 → 分润结算
  │        权限: 加盟商只看自己门店数据(RLS扩展)
  └─ 验收标准: Mac mini能同步 + 加盟商能独立登录管理门店
```

### Phase 2: 系统可卖（第5-10周）— 补齐竞争力

```
Week 5-6: 供应链深化
  ├─ P1-1: 中央厨房生产计划排程
  ├─ P1-2: 配送管理迁移到PG + 路线优化
  ├─ P1-7: 门店间调拨DB持久化 + 会计联动
  └─ P1-8: 供应商门户迁移

Week 7-8: 财务+合规
  ├─ P1-3: 预算管理（编制+对比+预警）
  ├─ P1-4: 电子发票对接（诺诺/百望云）
  └─ P1-5: 菜单模板PG持久化 + 多渠道发布

Week 9-10: 边缘AI + 新渠道
  ├─ P1-6: Core ML 3个模型训练部署
  ├─ P1-9: 抖音小程序编译 + 支付
  └─ 验收标准: 全功能Demo可向客户演示
```

### Phase 3: 系统有壁垒（第11-18周）— 构建差异化
