## 三、关键差距清单（按严重度排序）

### 🔴 P0 — 致命差距（影响 Tier 1 安全/资金路径）

#### G1. CRDT 冲突解决是伪 CRDT（LWW 策略）

**位置**：`edge/sync-engine/src/conflict_resolver.py`（仅 102 行）

**现状**："云端优先 + 本地终态保护"的简单策略，不是真正的 CRDT（Conflict-free Replicated Data Type）。

```python
# 当前逻辑：
# 1. 本地终态 + 时间戳更新 → 保留本地
# 2. 本地终态 vs 远端非终态 → 保留本地  
# 3. 默认 → 云端覆盖本地
```

**风险**：断网期间两端同时修改同一订单的不同字段（如：本地改了状态、云端改了金额），LWW 会丢失一方的修改。

**CLAUD.md 承诺**：Tier 1 路径标注"CRDT 冲突解析"，但实现不是 CRDT。

**建议**：实现真正的 CRDT（如状态用 LWW Map、金额用 PN-Counter、菜品用 OR-Set），或降级文档中的描述为"LWW + 终态保护"。

---

#### G2. mac-station 离线 API 存在显式 Stub

**位置**：
- `edge/mac-station/src/vision_stub.py`（95 行，标注 stub）
- `edge/mac-station/src/prediction_stub.py`（166 行，标注 stub）

**风险**：断网时 Mac mini 的 AI 推理（出餐时间预测、折扣异常检测）走 rule-based fallback，准确率远低于 CoreML 模型。

**Tier 1 断网 4 小时测试**：当前测试验证的是数据同步不丢失，未验证 AI 推理降级的影响。

**建议**：补全 CoreML 模型训练 + 部署流程，或明确文档化"离线模式降级为规则引擎"的 SLA。

---

#### G3. 边缘 CoreML Bridge 仍为 Demo 级

**位置**：`edge/coreml-bridge/`（Swift 706 行 + Python 643 行 = 1,349 行）

**现状**：
- Swift 端硬编码模型路径，无模型版本管理
- 加载 CoreML `.mlpackage`，但无 A/B 测试框架
- Python 端 `rule_fallback.py`（248 行）是主要推理路径
- 无模型性能监控/漂移检测

**影响**：CLAUD.md 描述的"边缘 AI 推理"核心竞争力未建立。

## 四、AI 代码质量关注点

### 增速与质量平衡

一个月内 Python 代码从 389K → 960K（净增 571K 行），TypeScript 从 80K → 500K+（净增 420K+ 行）。日均产出约 30K 行。

**典型 AI 生成模式特征（已观察到）**：
- 大量文件但单文件平均行数偏低
- 部分服务（tx-expense, tx-pay, tx-predict, tx-devforge）文件多但测试极少
- 484 个迁移版本中部分可能是骨架迁移（仅包含表结构无业务逻辑迁移）

### 建议的代码质量审计

1. **死代码扫描**：21 个服务中有多少路由/服务文件实际未被 main.py 注册
2. **重复代码检测**：相似业务逻辑在 tx-trade / tx-finance / tx-expense 之间可能有重复
3. **import 链分析**：服务间是否存在意外的紧耦合
4. **迁移 chain 完整性**：484 个版本中是否有断链

## 七、一句话结论

> **屯象OS 代码规模（~1.5M 行）已达到甚至超过企业级餐饮 SaaS。在交易收银、会员 CRM、组织人事、安全合规、旧系统适配等维度已生产可用或接近生产可用。但"Palantir 级 AI 操作系统"定位的关键支柱——边缘 AI 推理、真实 CRDT 同步、全端测试覆盖、实时运营决策——仍需 8-12 周高强度补齐。当前综合评分约 78/100，核心风险在边缘 AI 产品化程度和前端测试覆盖率两项。**
