## 第一章 总则

### 第一条 目的

建立屯象科技统一的密码管理规范，覆盖账号密码、API密钥、加密密钥、数据库凭证等所有类型的凭证信息，防范因密码安全问题导致的未授权访问和数据泄露事件，满足等保三级对"身份鉴别"和"数据保密性"控制点的要求。

### 第二条 依据

1. GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》第三级身份鉴别要求；
2. GB/T 25058-2019《信息安全技术 网络安全等级保护实施指南》；
3. 本公司《信息安全管理办法》（TXK-SEC-001）；
4. 本公司《网络安全管理规定》（TXK-SEC-002）。

### 第三条 适用范围

本制度适用于公司所有人员在使用公司信息系统时涉及的所有类型凭证，包括：
- 操作系统账号密码；
- 应用系统账号密码；
- 数据库账号密码；
- API密钥和Token；
- 加密密钥（对称/非对称）；
- 数字证书私钥；
- 云平台访问密钥。

## 第二章 账号密码规范

### 第四条 密码复杂度要求

所有账号密码（包括个人账号和系统账号）须满足以下强度要求：

| 要求项目 | 具体标准 | 适用场景 |
|---------|---------|---------|
| 最小长度 | ≥ 12个字符 | 所有账号 |
| 大写字母 | 至少包含1个（A-Z）| 所有账号 |
| 小写字母 | 至少包含1个（a-z）| 所有账号 |
| 数字 | 至少包含1个（0-9）| 所有账号 |
| 特殊字符 | 至少包含1个（!@#$%^&*()-_=+[]{}|;':,./<>?）| 所有账号 |
| 禁用模式 | 不得包含用户名、姓名拼音、公司名称、系统名称 | 所有账号 |
| 键盘序列禁止 | 禁止使用123456、qwerty、abcdef等顺序字符 | 所有账号 |
| 常见密码禁止 | 系统维护常见弱密码黑名单，禁止使用 | 所有账号 |

**高权限账号（管理员/root）额外要求**：
- 最小长度 ≥ 16个字符；
- 建议使用密码管理器生成随机密码（如Bitwarden、1Password）；
- 强制启用MFA（多因素认证）。

### 第五条 密码生命周期管理

**更换周期**：

| 账号类型 | 强制更换周期 | 备注 |
|---------|-----------|------|
| 普通账号 | 90天 | 系统自动提醒，到期须强制修改 |
| 管理员账号 | 90天 | 同上 |
| 服务账号（数据库等）| 90天 | 须配合应用更新，计划维护窗口执行 |
| 临时账号 | 任务结束后立即注销，无需更换 | — |

**历史密码限制**：
- 禁止重复使用最近5次使用过的密码；
- 系统须存储最近5次密码的哈希值用于校验。

**密码重置要求**：
- 初始分配的临时密码须在首次登录时强制修改；
- 密码找回须通过以下方式验证身份：手机短信验证码 + MFA验证码（双因素）；
- 管理员重置他人密码须记录操作日志，被重置人须在下次登录时强制修改。

### 第六条 登录失败锁定策略

| 系统 | 失败次数阈值 | 锁定时长 | 解锁方式 |
|------|-----------|---------|---------|
| 应用系统（前端）| 5次/30分钟 | 30分钟 | 超时自动解锁或管理员手动解锁 |
| JumpServer堡垒机 | 5次/30分钟 | 30分钟 | 仅管理员手动解锁 |
| 腾讯云控制台 | 按腾讯云策略 | — | 腾讯云内置策略 |
| 数据库管理账号 | 3次/10分钟 | 60分钟 | 仅管理员手动解锁 |

锁定事件须触发告警通知（企业微信/短信），由安全管理员确认是否为恶意攻击。

### 第七条 密码存储规范

**系统层面密码存储要求**：

1. **哈希算法要求**：用户密码存储须使用以下推荐算法之一：
   - Argon2id（首选，现代密码哈希标准）；
   - bcrypt（cost factor ≥ 12）；
   - PBKDF2（迭代次数 ≥ 600,000，使用SHA-256）。
   禁止使用MD5、SHA-1、SHA-256直接哈希密码（不加盐）。

2. **加盐要求**：每个密码必须使用唯一的随机盐值（salt），禁止所有用户共用一个盐值。

3. **禁止明文存储**：任何情况下禁止以明文形式存储密码，包括数据库、日志、配置文件、代码注释。

4. **禁止可逆加密存储**：密码不得使用可逆加密（如AES）存储，只能使用单向哈希。

## 第三章 API密钥与Token管理

### 第八条 API密钥存储要求

**强制要求**：

1. **禁止提交代码仓库**：API密钥、Secret、Token等凭证严禁出现在：
   - 代码文件（.py/.ts/.js等）；
   - 配置文件（.env/.yaml/.json/.toml等）；
   - 文档文件（.md/.txt等）；
   - Git历史提交记录中。

2. **git-secrets防护**：
   ```bash
   # 所有开发环境须安装并配置git-secrets
   git secrets --install
   git secrets --register-aws
   # 添加自定义模式
   git secrets --add 'sk-[a-zA-Z0-9]{40,}'  # Claude API Key
   git secrets --add 'AKID[A-Z0-9]{32}'      # 腾讯云AccessKey
   git secrets --add 'eyJ[a-zA-Z0-9_-]{20,}' # JWT Token
   ```

3. **合规存储位置**（按优先级排序）：
   - **首选**：腾讯云KMS（生产环境所有密钥）；
   - **次选**：Docker/systemd secrets（容器化部署场景）；
   - **允许**：服务器环境变量（须通过配置管理工具注入，不得写入磁盘文件）；
   - **测试环境**：`.env.local`文件（须加入.gitignore，不得提交）。

4. **禁止输出到日志**：应用代码须确保日志中不输出任何密钥信息，日志框架须配置敏感字段过滤（如：password, secret, key, token, authorization）。

### 第九条 API密钥轮换管理

**轮换周期**：

| 密钥类型 | 正常轮换周期 | 泄露处置要求 |
|---------|-----------|------------|
| Claude API Key（Anthropic）| 90天 | 立即轮换（1小时内）|
| 微信小程序AppSecret | 90天 | 立即轮换（1小时内）|
| 支付宝应用私钥 | 90天 | 立即轮换（1小时内）|
| JWT Access Token签名密钥 | 90天 | 立即轮换（须使所有在线Token失效）|
| JWT Refresh Token签名密钥 | 90天 | 立即轮换（须使所有在线Token失效）|
| 腾讯云子账号AccessKey | 90天 | 立即轮换（1小时内）|
| 短信服务密钥 | 90天 | 立即轮换（1小时内）|

**轮换流程**：

```
1. 提前7天创建新密钥（与旧密钥并行有效期）
   ↓
2. 在测试环境验证新密钥可正常使用
   ↓
3. 提交变更申请，计划维护窗口执行
   ↓
4. 生产环境更新密钥（通过KMS或环境变量）
   ↓
5. 验证使用新密钥的功能正常
   ↓
6. 撤销旧密钥
   ↓
7. 更新密钥轮换记录（记录轮换时间、执行人）
```

**泄露应急轮换**（发现密钥泄露时）：
1. 立即（1小时内）在密钥来源平台撤销泄露密钥；
2. 不等待测试，直接部署新密钥到生产环境；
3. 监控旧密钥的历史调用记录，评估损失；
4. 上报安全事件（参见TXK-SEC-005）。

## 第四章 数据库密码管理

### 第十条 数据库账号管理规范

**账号分类与权限**：

| 账号类型 | 用途 | 最大权限 | 密码要求 |
|---------|------|---------|---------|
| 应用只读账号 | 查询业务数据 | SELECT（特定表）| 16位随机密码，90天轮换 |
| 应用读写账号 | 业务CRUD操作 | SELECT/INSERT/UPDATE/DELETE（特定表）| 16位随机密码，90天轮换 |
| 迁移账号 | 执行数据库迁移 | DDL权限（特定条件）| 16位随机密码，仅迁移时临时启用 |
| DBA账号 | 数据库管理 | 全部权限 | 20位随机密码，MFA，90天轮换，仅通过堡垒机 |
| 备份账号 | 执行备份 | CONNECT + SELECT | 16位随机密码，90天轮换 |

**禁止事项**：
- 禁止应用程序使用具有超级用户（superuser）权限的账号连接数据库；
- 禁止在数据库连接字符串中硬编码密码（须通过KMS或环境变量获取）；
- 禁止在应用日志中输出数据库连接字符串；
- 禁止DBA账号直接从外网访问数据库（须通过JumpServer）。

### 第十一条 数据库连接安全配置

```python
# 正确的数据库连接方式（从环境变量/KMS获取凭证）

import os
from sqlalchemy import create_engine

# 从环境变量获取（通过systemd/Docker secrets注入，不写入文件）
DATABASE_URL = os.environ.get("DATABASE_URL")
if not DATABASE_URL:
    raise RuntimeError("DATABASE_URL environment variable not set")

# 确保使用TLS连接
engine = create_engine(
    DATABASE_URL,
    connect_args={
        "sslmode": "require",  # 强制SSL
        "sslrootcert": "/etc/ssl/certs/ca-certificates.crt",
    },
    pool_pre_ping=True,
    pool_size=10,
    max_overflow=20,
)

# 错误示例（严禁）：
# DATABASE_URL = "postgresql://user:hardcoded_password@host/db"  # ❌ 严禁
```

## 第五章 密码共享禁止条款

### 第十二条 禁止共享密码的场景

以下情形属于密码共享违规行为，一经发现按《信息安全管理办法》第五章处置：

1. **账号共享**：多人使用同一账号和密码登录系统；
2. **口头传递**：通过电话、面对面口述等方式向他人告知密码；
3. **明文传送**：通过微信、钉钉、邮件等即时通讯工具以明文发送密码；
4. **记录存储**：将密码记录在便签纸、Excel文件（未加密）、普通文本文件中；
5. **浏览器保存**：将生产系统密码保存在浏览器密码管理器（非公司认可的密码管理工具）中；
6. **跨环境使用**：生产环境密码与开发/测试环境使用相同密码。

### 第十三条 密码安全传递方式

当确实需要向他人（如新入职人员）传递初始密码时，须采用以下安全方式：

1. **首选**：通过密码管理系统（Bitwarden企业版等）安全共享，接收方须立即修改；
2. **次选**：通过加密通道（如Signal、企业微信一对一私聊）发送临时密码，发送后立即删除消息；
3. **禁止**：通过公开群聊、邮件、微信朋友圈等方式传递密码。

## 第六章 密码管理工具

### 第十四条 推荐密码管理工具

| 场景 | 推荐工具 | 备注 |
|------|---------|------|
| 个人账号密码管理 | Bitwarden（开源，可自托管）| 生成高强度随机密码，加密存储 |
| 团队密钥共享 | Bitwarden组织功能 | 共享密钥须有访问记录 |
| 生产环境密钥 | 腾讯云KMS | 所有生产密钥的权威存储 |
| 开发环境临时密钥 | .env.local（不提交Git）| 须加入.gitignore |

### 第十五条 密码管理工具使用要求

1. 密码管理工具（Bitwarden等）本身须使用强主密码（≥20位）+ MFA保护；
2. 密码管理工具的主密码不得存储在任何数字设备上（须记忆或纸质离线保存于保险箱）；
3. 离职人员须将工作相关密码从个人密码管理工具中删除，并上交公司相关账号凭证。

## 第七章 违规处置

### 第十六条 密码安全违规处置标准

| 违规行为 | 违规等级 | 处置措施 |
|---------|---------|---------|
| 密码复杂度不达标（首次发现）| 一级 | 警告，强制立即修改 |
| 密码超过更换周期未更换 | 一级 | 警告，系统强制修改 |
| 通过非加密渠道传递密码（无实质泄露）| 二级 | 警告+整改 |
| 密码写在便签纸/明文文档中 | 二级 | 警告+整改，立即销毁记录 |
| API密钥硬编码提交至代码仓库 | 二级 | 警告+整改，立即轮换密钥，清理Git历史 |
| 多人共用同一账号密码 | 二级 | 警告+整改，立即重置账号 |
| 故意将密码告知未授权人员 | 三级 | 停职调查 |
| 因密码管理不当导致系统被入侵 | 三至四级（依影响确定）| 停职调查或解除合作 |
