## 第一章 总则

### 第一条 目的

为规范屯象科技网络安全管理工作，保障屯象OS SaaS平台网络基础设施和通信链路的安全稳定运行，防范网络攻击、数据窃取和未授权访问等安全威胁，满足等保三级网络安全管理要求，特制定本规定。

### 第二条 依据

1. GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》第三级控制点；
2. 《网络安全法》第二十一条网络安全保护义务；
3. 腾讯云安全最佳实践指南；
4. 本公司《信息安全管理办法》（TXK-SEC-001）。

### 第三条 适用范围

本规定适用于公司所有网络资产的规划、建设、运维和管理，包括：
- 腾讯云广州地域生产网络环境；
- 开发、测试网络环境；
- 远程办公网络接入；
- 屯象OS各类API接口和微服务通信。

## 第二章 网络架构管理

### 第四条 三区隔离架构

屯象OS生产网络环境按照"三区分离"原则进行安全域划分，各区域通过安全组（Security Group）和VPC子网实现强制隔离。

```
┌─────────────────────────────────────────────────────────────┐
│                      腾讯云广州地域                           │
│                                                              │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  防御区（DMZ Zone）                                   │   │
│  │  • WAF（Web应用防火墙）                               │   │
│  │  • Anti-DDoS高防IP                                   │   │
│  │  • CLB（负载均衡）                                   │   │
│  │  • 对外暴露服务：HTTPS:443, WSS:443                   │   │
│  └──────────────────┬───────────────────────────────────┘   │
│                     │ 受控转发（仅允许特定端口）              │
│  ┌──────────────────▼───────────────────────────────────┐   │
│  │  服务区（Service Zone）                               │   │
│  │  • FastAPI应用服务器（多实例）                        │   │
│  │  • Redis缓存集群                                     │   │
│  │  • 微服务通信（内网端口，不对外暴露）                  │   │
│  │  • 开放端口：8000-8099（仅防御区访问）                │   │
│  └──────────────────┬───────────────────────────────────┘   │
│                     │ 最小化连接（仅数据库端口）              │
│  ┌──────────────────▼───────────────────────────────────┐   │
│  │  管理区（Management Zone）                            │   │
│  │  • PostgreSQL数据库主从集群                           │   │
│  │  • JumpServer堡垒机（SSH:22，仅管理区内访问）         │   │
│  │  • 腾讯云CLS日志服务接入点                            │   │
│  │  • KMS密钥管理服务接入点                             │   │
│  │  • 管理端口：5432, 6379（仅服务区访问）               │   │
│  └──────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 第五条 网络访问控制原则

1. **默认拒绝原则**：所有安全组规则默认拒绝一切流量，仅显式允许必要的端口和协议；
2. **最小开放原则**：仅开放业务必需的网络端口，未使用的端口一律关闭；
3. **方向控制原则**：入站和出站规则分别管理，出站流量也须受控；
4. **互联网访问限制**：服务区和管理区服务器禁止直接访问互联网，如需更新软件包，通过NAT网关或内网代理访问；
5. **跨区通信审批**：任何跨安全域的新建通信路径，须经安全管理员审批，提交变更申请。

### 第六条 端口管理规范

**对外开放端口（防御区向互联网）**：

| 端口 | 协议 | 用途 | 备注 |
|------|------|------|------|
| 443 | HTTPS/WSS | API服务、WebSocket实时通信 | 必须，通过WAF过滤 |
| 80 | HTTP | 强制跳转HTTPS | 仅允许301重定向，禁止业务流量 |

禁止对外开放：22（SSH）、3306（MySQL）、5432（PostgreSQL）、6379（Redis）及其他管理端口。

**内部通信端口（服务区内部）**：

| 端口范围 | 用途 | 访问来源 |
|---------|------|---------|
| 8000-8099 | FastAPI微服务 | 防御区CLB + 服务区内部 |
| 5432 | PostgreSQL | 仅服务区应用服务器 |
| 6379 | Redis | 仅服务区应用服务器 |
| 22 | SSH（JumpServer） | 仅管理区内堡垒机出口 |

### 第七条 网络安全设备管理

1. **WAF策略管理**：WAF规则集每季度审查一次，发现新型Web攻击特征及时更新规则；业务上线前须验证WAF规则不影响正常功能；
2. **Anti-DDoS配置**：DDoS防护阈值根据业务实际流量设置，阈值触发后自动告警；
3. **CLB健康检查**：负载均衡健康检查配置须与实际服务健康接口保持一致；
4. **安全组审查**：每季度审查所有安全组规则，清理无效或冗余规则，审查记录归档保留。

## 第三章 账号密码管理规范

### 第八条 密码复杂度要求

所有系统账号密码须满足以下复杂度要求：

| 要求项 | 标准 |
|-------|------|
| 最小长度 | 12位字符 |
| 大写字母 | 至少包含1个（A-Z） |
| 小写字母 | 至少包含1个（a-z） |
| 数字 | 至少包含1个（0-9） |
| 特殊字符 | 至少包含1个（!@#$%^&*等） |
| 禁用字符组合 | 禁止使用用户名、公司名、系统名作为密码的一部分 |
| 常见密码 | 禁止使用已知常见密码（如Password123!） |

### 第九条 密码生命周期管理

1. **更换周期**：所有账号密码须每90天强制更换一次；
2. **历史密码限制**：禁止重复使用最近5次使用过的密码；
3. **初始密码**：新建账号初始密码须在首次登录时强制修改；
4. **临时密码**：为外部人员创建的临时账号密码，访问结束后立即注销账号；
5. **密码找回**：密码找回须经过身份验证（手机验证码+MFA），记录密码重置操作日志。

### 第十条 账号管理规范

1. **禁止共享账号**：每个人员对应唯一账号，禁止多人共用一个账号；
2. **禁止明文存储**：密码禁止以明文形式存储，系统内存储密码必须使用bcrypt、Argon2等不可逆哈希算法；
3. **失败锁定**：连续5次登录失败后，账号锁定30分钟，解锁须人工干预或等待超时；
4. **闲置账号处理**：连续90天未登录的账号自动禁用，需经管理员审批重新激活；
5. **离职账号处置**：员工离职或合作终止后，24小时内完成账号禁用，权限回收记录归档。

### 第十一条 多因素认证要求

以下系统和操作必须启用多因素认证（MFA）：

| 系统/场景 | MFA要求 | 实现方式 |
|---------|---------|---------|
| 腾讯云控制台 | 强制 | 腾讯云MFA（TOTP） |
| JumpServer堡垒机 | 强制 | TOTP + 短信双因素 |
| 生产数据库（直连） | 强制 | 证书 + 密码双因素 |
| GitHub代码仓库 | 强制 | GitHub MFA（TOTP） |
| 管理员账号登录 | 强制 | TOTP |
| 普通用户登录 | 推荐 | TOTP |

## 第四章 远程访问管理

### 第十二条 堡垒机访问规范

所有对生产服务器的运维操作必须通过JumpServer堡垒机进行，禁止任何形式的直接SSH访问生产服务器。

**堡垒机基本配置要求**：
1. 堡垒机部署在管理区，仅向运维人员的固定IP开放22端口；
2. 运维人员须使用个人账号登录堡垒机，禁止使用共享账号；
3. 所有通过堡垒机的操作自动录屏，录屏文件保留180天；
4. 堡垒机本身须启用MFA登录；
5. 堡垒机账号权限与实际服务器账号权限严格对应，最小权限原则。

**操作前置检查**：
1. 变更类操作须先完成变更申请（参见《信息系统运维安全管理规定》）；
2. 登录后第一步：确认当前连接的服务器名称和环境（开发/测试/生产），避免误操作；
3. 高危操作（删除数据、停服务、修改配置）须在操作前通过企业微信发送操作计划，留存记录。

### 第十三条 远程办公安全要求

1. **网络要求**：使用公共Wi-Fi时，必须通过公司VPN或直接使用JumpServer堡垒机，禁止在未加密网络中访问生产系统；
2. **设备要求**：访问生产系统的设备须安装最新操作系统安全补丁，启用全盘加密（FileVault/BitLocker）；
3. **屏幕锁定**：工作设备在闲置5分钟后自动锁屏；
4. **会话超时**：堡垒机SSH会话闲置30分钟后自动断开；
5. **敏感操作地点限制**：涉及生产数据库直接操作的高危运维，须在安全可信的网络环境中进行。

## 第五章 漏洞管理流程

### 第十四条 漏洞扫描要求

| 扫描类型 | 频率 | 工具 | 执行人 |
|---------|------|------|-------|
| 应用层漏洞扫描（SAST） | 每次代码提交 | bandit（Python）, ESLint安全规则 | CI/CD自动触发 |
| 依赖组件漏洞扫描（SCA） | 每次构建 | trivy, Safety | CI/CD自动触发 |
| 基础设施漏洞扫描 | 每季度 | 腾讯云安全扫描服务 | 安全管理员 |
| Web应用渗透测试 | 每半年 | 专业渗透测试工具 | 安全管理员/外包 |
| 端口和服务扫描 | 每季度 | nmap + 腾讯云暴露面检测 | 安全管理员 |

### 第十五条 漏洞修复时限

根据漏洞CVSS评分或公司内部评级，执行以下修复时限要求：

| 漏洞等级 | 判定标准 | 修复时限 | 修复验证 |
|---------|---------|---------|---------|
| 紧急（Critical） | CVSS≥9.0 或 正在被利用 | 24小时内修复或临时缓解 | 修复后立即验证并报告 |
| 高危（High） | CVSS 7.0-8.9 | 48小时内修复 | 修复后24小时内验证 |
| 中危（Medium） | CVSS 4.0-6.9 | 7个工作日内修复 | 修复后3日内验证 |
| 低危（Low） | CVSS<4.0 | 下一个常规迭代版本修复 | 版本发布前验证 |

**修复流程**：
1. 漏洞发现 → 记录漏洞信息（编号/描述/影响范围/CVSS评分）；
2. 安全管理员评估漏洞影响，确定修复优先级；
3. 高危及以上漏洞立即启动应急响应（必要时临时下线或隔离受影响服务）；
4. 开发/运维完成修复，在测试环境验证有效性；
5. 发布到生产环境（高危以上须额外提交变更申请）；
6. 修复后验证，确认漏洞已消除；
7. 更新漏洞记录，归档处置过程。

### 第十六条 漏洞信息保密

1. 漏洞信息属于公司机密，禁止向未授权人员披露；
2. 漏洞修复前，禁止在公开渠道讨论漏洞细节；
3. 外部渗透测试报告须加密传输和存储，严格限制查阅范围；
4. 修复后6个月内，漏洞详情仍属内部保密信息。

## 第六章 API密钥管理

### 第十七条 API密钥存储规范

1. **禁止提交代码仓库**：API密钥、Token、密码等敏感信息严禁提交至Git仓库（含私有仓库）；
2. **git-secrets防护**：所有开发环境须安装git-secrets工具，自动拦截包含敏感信息的代码提交；
3. **存储方式**：API密钥须存储于以下安全位置之一：
   - 腾讯云KMS（生产环境，强烈推荐）；
   - 服务器环境变量（通过systemd或Docker secrets注入）；
   - 腾讯云SSM参数仓库（次选）；
4. **禁止硬编码**：禁止在代码文件、配置文件、SQL文件、Shell脚本等任何文件中硬编码密钥；
5. **禁止日志记录**：确保应用日志中不输出API密钥、密码、Token等敏感信息。

### 第十八条 API密钥轮换周期

| 密钥类型 | 轮换周期 | 轮换触发条件 |
|---------|---------|------------|
| 外部API密钥（Claude API、微信API等） | 90天 | 到期或泄露 |
| JWT签名密钥 | 90天 | 到期或泄露 |
| 数据库密码 | 90天 | 到期或泄露 |
| 腾讯云子账号AccessKey | 90天 | 到期或泄露 |
| TLS证书 | 90天或到期前30天 | 到期前自动续签 |
| 数据加密密钥（DEK） | 1年 | 到期或泄露 |

**泄露应急处置**：
发现密钥可能泄露时（包括但不限于代码仓库历史提交、错误日志输出、权限异常访问），须：
1. **立即**（1小时内）在密钥来源平台撤销泄露密钥；
2. 生成新密钥并更新所有使用方；
3. 审查泄露密钥的历史访问记录，评估数据损失；
4. 上报安全事件，启动相应级别的应急响应；
5. 归档事件处置过程。

## 第七章 安全事件上报流程

### 第十九条 网络安全事件分类

本章针对网络安全相关事件的上报，完整应急响应流程见《安全事件分类分级与应急预案》（TXK-SEC-005）。

网络安全相关事件包括：
- 疑似DDoS攻击（流量异常、服务响应超时）；
- Web应用攻击告警（WAF触发大量拦截、SQL注入、XSS攻击）；
- 未授权访问尝试（多次登录失败告警、异常IP访问）；
- API密钥泄露或异常调用；
- 网络端口扫描或探测行为；
- 内网横向移动迹象；
- DNS劫持或流量劫持迹象。

### 第二十条 网络安全事件上报流程

```
发现异常
    │
    ▼（30分钟内）
记录事件基本信息
（时间/来源IP/影响服务/告警内容）
    │
    ▼
评估影响范围
    │
    ├── 影响生产服务 ──► 立即启动应急响应（见TXK-SEC-005）
    │                    同时通知：腾讯云安全应急 95716
    │
    └── 未影响生产  ──► 4小时内上报安全管理员
                        安全管理员决定是否升级处理
```

### 第二十一条 安全告警配置要求

以下安全事件须配置自动告警，告警通知至负责人手机（企业微信/短信）：

| 告警类型 | 触发条件 | 响应时限 |
|---------|---------|---------|
| 登录失败 | 同一账号5分钟内失败5次 | 30分钟内确认 |
| 异常登录 | 异常地理位置或时间登录 | 30分钟内确认 |
| DDoS触发 | Anti-DDoS防护阈值触发 | 立即响应 |
| WAF高危拦截 | 高危攻击类型拦截（SQL注入等） | 1小时内确认 |
| 证书即将过期 | 提前30天告警 | 7天内续签 |
| 磁盘使用率 | 超过80% | 24小时内扩容 |
| 异常流量 | 流量超过基线3倍 | 30分钟内确认 |
