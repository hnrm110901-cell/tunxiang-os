## 第一章 总则

### 第一条 目的

规范屯象科技与外部供应商、合作伙伴及外包人员之间的访问管理流程，确保第三方访问行为可控、可追溯，防范因外部人员访问引发的数据泄露和系统安全事件。

> **编写背景**：根据同类SaaS产品等保测评报告（参考测评报告）显示，"外部人员未签署保密协议即可访问系统"是导致扣分的典型问题。本规定针对性解决该问题，建立保密协议前置审签机制和全程访问监控体系，确保等保三级测评顺利通过。

### 第二条 依据

1. GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》第三级外部人员访问管理；
2. 《数据安全法》第三十二条数据安全义务；
3. 《个人信息保护法》第二十三条数据委托处理要求；
4. 本公司《信息安全管理办法》（TXK-SEC-001）；
5. 本公司《信息系统运维安全管理规定》（TXK-SEC-004）。

### 第三条 适用范围

本规定适用于以下所有需要访问屯象科技信息系统或数据的外部人员：

| 人员类型 | 说明 |
|---------|------|
| 技术外包人员 | 受雇于外包公司、协助开发或运维的技术人员 |
| 安全测评机构人员 | 等保测评机构、渗透测试人员 |
| 审计人员 | 外部审计机构、会计师事务所人员 |
| 云服务商技术支持 | 腾讯云等云服务商的技术支持工程师 |
| 第三方系统集成商 | 与屯象OS进行系统对接的第三方技术人员 |
| 实习生 | 非正式员工的实习人员 |
| 其他访客 | 临时需要访问公司系统的外部人员 |

## 第二章 供应商安全评估

### 第四条 采购前安全评估要求

在选择或采购以下类型的供应商/服务之前，须完成安全评估，评估结果归档保存：

**须评估的供应商类型**：
- 云基础设施服务商（IaaS/PaaS）；
- 第三方API服务（支付、短信、地图、AI等）；
- 开发工具和DevOps工具（CI/CD、代码仓库等）；
- 数据处理相关服务（日志分析、数据库托管等）；
- 技术外包和IT服务供应商。

### 第五条 供应商安全评估清单

在确定供应商前，须完成以下评估：

**资质评估**：
- [ ] 1. 供应商是否持有相关安全认证（ISO 27001、等保证书等）；
- [ ] 2. 供应商是否有清晰的信息安全政策文档；
- [ ] 3. 供应商是否经历过重大安全事件，如何处置；
- [ ] 4. 供应商是否有数据泄露通知机制（发生泄露时是否会及时通知客户）。

**合同安全条款评估**：
- [ ] 5. 合同中是否明确数据所有权（公司数据属于屯象科技）；
- [ ] 6. 合同中是否有保密条款；
- [ ] 7. 合同中是否明确数据处理地域限制（数据存储在中国境内）；
- [ ] 8. 合同中是否有安全事件通报义务条款；
- [ ] 9. 合同中是否有服务终止后的数据删除/归还条款；
- [ ] 10. 供应商是否允许审计其安全措施。

**技术安全评估**：
- [ ] 11. 供应商数据传输是否加密（TLS 1.2+）；
- [ ] 12. 供应商存储的敏感数据是否加密；
- [ ] 13. 供应商是否有访问日志和审计机制；
- [ ] 14. 供应商是否有漏洞管理和安全更新机制。

**评估结论**：□ 通过，可以采购  □ 有条件通过（须满足以下条件：___________）□ 不通过

## 第三章 外部人员访问流程

### 第六条 访问申请流程

```
申请阶段（访问前至少提前1个工作日）
    │
    ▼
1. 提交外部人员访问申请表（见附件一）
   填写内容：访问目的/所需权限/访问系统/访问时间段/人员信息
    │
    ▼
2. 外部人员签署《信息安全保密协议》（见附件二）
   【强制前置条件：未签署保密协议，不得授予任何访问权限】
    │
    ▼
3. 李淳审批
   通过企业微信确认，留存审批截图
    │
    ▼
4. 创建临时访问账号
   命名规范：temp-[姓名缩写]-[YYYYMMDD]
   配置最小权限，设置账号有效期
    │
    ▼
访问实施阶段
    │
    ▼
5. 告知操作规范
   - 说明可访问的范围和禁止事项
   - 要求通过堡垒机访问（自动录屏）
   - 高风险操作须屯象科技人员全程陪同
    │
    ▼
6. 执行访问任务（全程录屏）
    │
    ▼
访问结束阶段
    │
    ▼
7. 任务完成后，外部人员退出系统
    │
    ▼
8. 24小时内注销临时账号
   删除账号，撤销所有权限
    │
    ▼
9. 审查操作日志（确认无异常操作）
    │
    ▼
10. 归档访问记录
    申请表+审批截图+保密协议+操作日志
```

### 第七条 不同场景访问权限标准

| 访问场景 | 权限范围 | 陪同要求 | 录屏要求 |
|---------|---------|---------|---------|
| 等保测评机构测评 | 只读访问（指定系统）+ 问卷填写 | 全程陪同 | 必须 |
| 渗透测试 | 指定攻击范围内的测试权限 | 可远程监控 | 必须 |
| 腾讯云技术支持 | 只读访问（通过腾讯云工单机制）| 授权后监控 | 通过腾讯云日志 |
| 第三方系统集成联调 | API联调专用测试账号（沙箱环境）| 无需（沙箱环境）| 建议 |
| 外包开发人员 | 代码仓库访问（无生产环境权限）| 无需 | Git操作记录 |
| 外包运维支持 | 堡垒机有限权限（指定服务器）| 必要时陪同 | 必须 |
| 审计人员 | 审计日志只读 | 无需 | 审计日志记录 |

### 第八条 外部访问关键禁止事项

外部人员访问期间，以下行为严格禁止：

1. **禁止数据导出**：禁止将任何生产数据、客户信息、代码等导出到外部存储介质或个人设备；
2. **禁止截图录屏**：未经授权，禁止截取包含客户数据、系统架构的屏幕内容；
3. **禁止账号转让**：禁止将临时账号信息告知其他未授权人员使用；
4. **禁止超权操作**：禁止任何超越申请权限范围的操作；
5. **禁止安装软件**：禁止在公司服务器上安装任何未经批准的软件；
6. **禁止留后门**：禁止在系统中创建任何隐藏账号或后门程序。

违反上述禁止事项，立即终止访问，按《信息安全管理办法》处置，情节严重者向公安机关报案。

### 第九条 访问记录保留要求

所有外部人员访问记录须保留至少**2年**，包括：
- 访问申请表（原件或扫描件）；
- 保密协议签署件；
- 李淳审批截图（企业微信）；
- 账号创建/注销记录；
- JumpServer操作录屏或访问日志；
- 访问结果确认记录。

## 第四章 保密协议要求

### 第十条 保密协议签署要求

1. **强制前置**：所有外部人员在获得任何访问权限前，须签署保密协议。这是硬性前置条件，不得因任何理由跳过；
2. **纸质签署**：原则上须签署纸质版保密协议，扫描存档；无法见面签署时，须使用具有法律效力的电子签名（如法大大、e签宝）；
3. **内容覆盖**：保密协议须覆盖所有可能接触的信息类型（系统架构、客户数据、商业秘密等）；
4. **有效期约定**：保密义务须约定明确期限，建议不短于业务关系终止后3年；
5. **违约条款**：明确违约责任，包括赔偿条款和法律追责条款。

## 第五章 云服务商责任边界

### 第十一条 腾讯云IaaS安全责任分工

屯象OS使用腾讯云广州地域作为主要基础设施平台，安全责任按照"共担责任模型"明确划分：

**腾讯云负责的安全范围**（通过腾讯云云安全责任声明和SLA承诺）：

| 安全层 | 腾讯云责任 | 依据 |
|-------|---------|------|
| 物理安全 | 数据中心访问控制、视频监控、防灾措施 | 腾讯云等保三级合规证书 |
| 硬件安全 | 服务器、存储、网络设备安全 | 腾讯云安全白皮书 |
| 虚拟化安全 | Hypervisor安全隔离、内核加固 | 腾讯云共担责任说明 |
| 网络基础设施 | 骨干网络安全、底层DDoS防护 | 腾讯云Anti-DDoS SLA |
| 密钥存储安全 | KMS硬件安全模块（HSM）物理安全 | 腾讯云KMS白皮书 |
| 服务高可用 | CDB主从复制、CVM可用区冗余 | 腾讯云SLA承诺 |

**屯象科技负责的安全范围**（屯象科技自行承担）：

| 安全层 | 屯象科技责任 | 管理措施 |
|-------|-----------|---------|
| 操作系统安全 | CVM内OS配置、补丁更新 | 定期更新，最小服务原则 |
| 应用程序安全 | FastAPI应用、数据库配置、API安全 | 安全开发规范、代码审计 |
| 数据加密 | 应用层加密、字段级加密 | AES-256加密，KMS管理 |
| 身份认证 | 应用层用户认证、JWT管理 | MFA、JWT轮换 |
| 访问控制 | 安全组规则、RLS策略、API权限 | 最小权限原则 |
| 合规管理 | 等保定级备案、整改 | 本系列制度文档 |
| 数据分类 | 数据分级、PII处理 | 数据分类分级制度 |
| 安全监控 | 应用层安全监控、日志分析 | CLS + 自建监控 |

### 第十二条 云服务合同安全条款确认

在签署腾讯云服务协议时，须确认以下安全条款：
- [ ] 数据归属条款（客户数据属于屯象科技所有）；
- [ ] 数据存储地域（数据存储在中国境内）；
- [ ] 数据泄露通知义务（腾讯云发生可能影响客户数据的安全事件时须通知）；
- [ ] 服务终止后数据处理条款（合同终止后数据如何归还/删除）；
- [ ] 安全审计权利（屯象科技有权审查腾讯云的安全合规状态）。

## 附件二：信息安全保密协议模板

```
信息安全保密协议

协议编号：TXK-NDA-[YYYYMMDD]-[序号]
签署日期：_______________

甲方：屯象科技（湖南）有限公司
代表人：李淳
地址：湖南省长沙市 [详细地址]

乙方（保密义务人）：
姓名：_______________
证件类型及号码：身份证 _______________
所属单位：_______________
联系方式：_______________

鉴于甲乙双方因 [合作事项简述] 需要乙方访问甲方的信息系统及数据，
经双方协商一致，就保密事项达成如下协议：

第一条 保密范围

乙方承诺对以下信息严格保密（以下统称"保密信息"）：
1. 甲方的系统架构设计、源代码、技术文档；
2. 甲方客户的个人信息（含姓名、手机号、消费记录等）；
3. 甲方的商业计划、运营数据、财务数据；
4. 甲方的安全漏洞信息、系统配置信息；
5. 乙方在合作过程中知悉的一切属于甲方商业秘密的信息；
6. 本协议的存在及内容本身。

第二条 保密义务

1. 乙方不得将保密信息泄露给任何第三方（包括乙方所属单位的其他人员，除非获得甲方明确书面授权）；
2. 乙方不得利用保密信息为自己或第三方谋取利益；
3. 乙方不得复制、录制、截图保密信息，除非明确获得甲方授权；
4. 乙方须按照甲方规定的操作规范访问系统，不得超越授权范围操作；
5. 乙方的访问操作须通过甲方堡垒机进行，接受自动录屏监控；
6. 协议期满或合作终止后，乙方须立即停止使用保密信息，并应甲方要求归还或销毁一切载有保密信息的材料。

第三条 保密期限

本协议保密义务自乙方首次接触保密信息之日起，至合作关系终止后 **三（3）年** 内持续有效。

第四条 违约责任

1. 乙方违反本协议，导致甲方遭受损失的，须赔偿甲方的全部直接损失；
2. 乙方违反本协议，导致甲方客户个人信息泄露的，须连带承担相应法律责任；
3. 乙方恶意泄露保密信息构成犯罪的，甲方有权向公安机关报案追究刑事责任；
4. 本条约定的违约责任不影响甲方寻求其他法律救济。

第五条 其他条款

1. 本协议依据中华人民共和国法律缔结和解释；
2. 因本协议产生的争议，双方协商解决；协商不成，向甲方住所地人民法院提起诉讼；
3. 本协议一式两份，甲乙双方各执一份，具有同等法律效力。

甲方代表签字：_______________  日期：_______________
（李淳，屯象科技创始人）

乙方签字：_______________  日期：_______________

（当无法签署纸质协议时，须使用法大大或e签宝等具有法律效力的电子签名平台签署）
```
