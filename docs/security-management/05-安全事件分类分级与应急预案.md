## 第一章 总则

### 第一条 目的

建立屯象OS安全事件分类分级标准和应急响应体系，确保在发生安全事件时能够快速、有序地响应和处置，最大限度降低事件对业务和客户的影响，满足等保三级对"安全事件处置"和"应急预案管理"控制点的要求。

### 第二条 依据

1. GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》第三级安全管理要求；
2. GB/T 20984-2022《信息安全技术 信息安全风险评估方法》；
3. 《国家网络安全事件应急预案》；
4. 《网络安全法》第五章关于监测预警与应急处置的规定；
5. 本公司《信息安全管理办法》（TXK-SEC-001）。

### 第三条 适用范围

本预案适用于屯象OS SaaS平台及其支撑基础设施发生的所有信息安全事件，包括数据泄露、系统入侵、服务中断、DDoS攻击、内部违规等。

## 第二章 安全事件分类分级

### 第四条 事件分级标准

依据事件影响范围、数据泄露量、服务中断时长等维度，安全事件分为四个等级：

#### I级：特别重大安全事件（红色）

满足以下任意一项即判定为I级：
- 客户PII数据泄露数量 ≥ 10万条；
- 交易数据或财务数据大规模泄露（涉及金额 ≥ 100万元）；
- 屯象OS核心服务全面瘫痪，持续时间 > 8小时；
- 系统遭受高级持续性威胁（APT）攻击，攻击者长期驻留；
- 勒索病毒加密核心数据库，业务无法正常运行；
- 事件已引起监管机关介入调查或媒体曝光。

**响应时限**：发现后**15分钟**内启动响应，**1小时**内完成初步隔离。

#### II级：重大安全事件（橙色）

满足以下任意一项即判定为II级：
- 客户PII数据泄露数量 ≥ 1000条且 < 10万条；
- 核心业务功能（收银/点餐/结账）中断 > 2小时；
- 内部员工权限被盗用，访问了大量非授权数据；
- API密钥或JWT密钥泄露，存在被滥用风险；
- 数据库遭受SQL注入攻击且成功执行了恶意查询；
- 系统遭受DDoS攻击导致服务不可用 > 1小时。

**响应时限**：发现后**30分钟**内启动响应，**2小时**内完成初步处置。

#### III级：较大安全事件（黄色）

满足以下任意一项即判定为III级：
- 单个租户数据泄露（客户会员数据、交易记录）；
- 业务功能降级（部分功能不可用，核心流程仍可运行）；
- 外部入侵尝试成功进入系统但未获取敏感数据；
- 单个服务或接口故障导致部分租户业务受影响 > 30分钟；
- 发现未授权的内部数据访问行为（但数据未外泄）；
- 开发/测试环境数据泄露（含生产数据镜像）。

**响应时限**：发现后**1小时**内启动响应，**4小时**内完成处置。

#### IV级：一般安全事件（蓝色）

满足以下任意一项即判定为IV级：
- 可疑入侵尝试（端口扫描、暴力破解攻击，未成功）；
- 单个接口故障（影响单一功能，持续时间 < 30分钟）；
- 员工违规操作（未遵守操作规范，无数据泄露）；
- 钓鱼邮件攻击（员工收到，未点击/已上报）；
- 系统配置错误（自动告警发现，无业务影响）；
- 第三方组件发现新漏洞（尚未影响生产）。

**响应时限**：发现后**4小时**内记录和评估，**24小时**内制定处置方案。

## 第三章 应急响应流程

### 第五条 通用响应流程

所有等级的安全事件均遵循以下五阶段响应流程：

```
┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐
│  发现  │→  │  上报  │→  │  隔离  │→  │  恢复  │→  │  复盘  │
└──────┘    └──────┘    └──────┘    └──────┘    └──────┘
 30分钟内    按级别       控制损失    业务恢复    48小时内
 完成上报    时限内       扩大范围    正常运营    提交报告
```

### 第六条 I级特别重大事件响应流程

```
发现事件
    │
    ▼（立即，<5分钟）
电话通知李淳（不论何时）
    │
    ▼（<15分钟）
成立紧急响应小组
李淳担任响应总指挥
    │
    ├──► 技术处置组（李淳）
    │    - 立即隔离受感染服务器（腾讯云控制台切断网络）
    │    - 保留现场证据（内存转储、日志备份）
    │    - 切换到备用服务或维护模式
    │
    ├──► 外部协同（<1小时）
    │    - 拨打腾讯云安全应急响应：95716
    │    - 根据事件性质，决定是否报告网络安全主管部门
    │
    └──► 客户通知（<2小时，如有数据泄露）
         - 制定客户通知方案
         - 必要时向监管机关报告（个人信息保护法要求）

持续阶段（每小时更新事件状态记录）
    │
    ▼
业务恢复
    │
    ▼（恢复后48小时内）
事件复盘报告
```

**I级事件强制通报要求**：
- 涉及客户PII数据泄露且人数超过10万人，须依据《个人信息保护法》向国家网信办报告；
- 涉及国家安全或社会稳定的事件，须向公安机关网络安全部门报告。

### 第七条 II级重大事件响应流程

**发现-上报（<30分钟）**：
1. 发现人通过企业微信立即通知李淳，说明事件类型和初步判断；
2. 李淳确认事件等级，启动II级响应；
3. 通知腾讯云客户经理（视情况）。

**隔离-控制（<2小时）**：
1. 隔离受影响服务或租户（腾讯云安全组快速封锁）；
2. 阻断攻击来源IP（WAF黑名单）；
3. 轮换可能已泄露的API密钥和Token；
4. 保全证据（日志、网络流量记录）。

**恢复（<4小时）**：
1. 漏洞修复或临时缓解；
2. 逐步恢复受影响服务；
3. 验证系统安全性后全量恢复；
4. 监控恢复后的系统状态至少24小时。

### 第八条 III级较大事件响应流程

**发现-上报（<1小时）**：
1. 发现人通过企业微信上报，提供事件描述和初步分析；
2. 安全管理员（李淳）4小时内确认响应方案。

**处置（<4小时）**：
1. 定位受影响范围（具体租户、数据类型、时间窗口）；
2. 实施临时控制措施（禁用异常账号、封锁异常IP）；
3. 修复漏洞或配置错误；
4. 验证修复效果。

**后续跟进**：
1. 受影响租户通知（如有数据泄露风险）；
2. 7天内提交事件报告。

### 第九条 IV级一般事件响应流程

1. 发现人记录事件详情，4小时内通过企业微信上报；
2. 安全管理员在24小时内完成评估和处置方案；
3. 纳入常规漏洞/问题跟踪，按正常迭代节奏修复；
4. 月度安全审计时汇总IV级事件清单。

### 第十一条 应急工具箱

以下命令用于安全事件快速排查，所有操作须通过JumpServer堡垒机执行：

**网络排查命令**：
```bash
# 查看当前活跃连接（快速发现异常连接）
ss -tnp | grep ESTABLISHED

# 查看监听端口（检查是否有未授权的监听服务）
ss -tlnp

# 查看最近的网络连接（按连接数统计来源IP）
netstat -tn | awk '{print $5}' | cut -d: -f1 | sort | uniq -c | sort -rn | head -20

# 检查ARP缓存（检测ARP欺骗）
arp -n

# 快速封锁攻击IP（腾讯云安全组或本地iptables）
iptables -A INPUT -s [攻击IP] -j DROP
```

**进程与账号排查命令**：
```bash
# 查看当前登录用户（发现异常会话）
w
who
last | head -30

# 查看异常进程（CPU/内存占用异常）
ps aux --sort=-%cpu | head -20
ps aux --sort=-%mem | head -20

# 查找最近修改的文件（检测是否有新增恶意文件）
find /var /tmp /usr /home -mtime -1 -type f 2>/dev/null | head -50

# 检查计划任务（排查持久化后门）
crontab -l
ls -la /etc/cron.d/ /etc/cron.hourly/ /etc/cron.daily/
```

**PostgreSQL数据库排查命令**：
```bash
# 查看当前活跃连接（检测异常查询）
psql -h [DB_HOST] -U postgres -c "
  SELECT pid, usename, application_name, client_addr,
         state, query_start, query
  FROM pg_stat_activity
  WHERE state = 'active'
  ORDER BY query_start;"

# 查询最近的登录审计日志（需提前配置log_connections）
grep "connection" /var/log/postgresql/postgresql-*.log | tail -100

# 检查是否存在超权用户
psql -h [DB_HOST] -U postgres -c "
  SELECT rolname, rolsuper, rolcreatedb, rolcreaterole
  FROM pg_roles
  WHERE rolsuper = true OR rolcreatedb = true;"

# 检查RLS策略是否生效（等保三级关键检查）
psql -h [DB_HOST] -U postgres -c "
  SELECT tablename, policyname, permissive, roles, qual
  FROM pg_policies
  ORDER BY tablename;"
```

**日志分析命令**：
```bash
# 查看Nginx/API访问日志中的异常请求（按IP统计）
awk '{print $1}' /var/log/nginx/access.log | sort | uniq -c | sort -rn | head -20

# 查找SQL注入特征
grep -i "union.*select\|or.*1=1\|drop.*table\|insert.*into" /var/log/nginx/access.log | tail -50

# 查看失败登录记录
grep "Failed password\|authentication failure" /var/log/auth.log | tail -50

# 腾讯云CLS日志查询（通过CLS控制台或CLI）
# 关键词：ERROR, CRITICAL, authentication failed, permission denied
```

**快速隔离命令**：
```bash
# 腾讯云CLI：快速修改安全组（封锁入站流量）
# tccli cvm ModifySecurityGroupsOfInstance --InstanceIds [...] --...

# Docker容器紧急停止
docker stop [container_name]

# 快速切换流量到维护页（修改Nginx配置）
# 将upstream指向维护页静态文件
nginx -t && nginx -s reload

# 数据库连接数限制（防止攻击者持续连接）
psql -c "ALTER USER [user] CONNECTION LIMIT 0;"
```

## 第五章 事后复盘要求

### 第十二条 事件报告提交时限

| 事件等级 | 事件报告提交时限 |
|---------|--------------|
| I级（特别重大） | 恢复后24小时内提交初步报告，72小时内提交完整报告 |
| II级（重大） | 恢复后48小时内提交完整报告 |
| III级（较大） | 恢复后7天内提交报告 |
| IV级（一般） | 纳入月度安全报告汇总 |

### 第十三条 事件复盘内容要求

事件报告须包含以下内容：

1. **事件概述**：发生时间、发现时间、影响范围、事件类型；
2. **事件经过**：详细时间线，从发现到恢复的完整过程；
3. **根本原因分析**：导致事件发生的根本技术原因和管理原因；
4. **影响评估**：数据损失量、业务中断时长、经济损失估算、受影响租户数量；
5. **处置措施**：已采取的临时缓解措施和永久修复措施；
6. **改进计划**：防止类似事件再次发生的系统性改进措施，附责任人和截止日期；
7. **经验总结**：响应过程中的经验和教训。

### 第十四条 复盘会议要求

I级和II级事件须在恢复后72小时内召开复盘会议（单人团队可通过录制视频形式记录），复盘记录归入安全事件档案。
