## 第一章 总则

### 第一条 目的

建立屯象OS信息资产完整清单，对所有信息资产进行分类分级管理，明确各类资产的保护要求和责任人，为信息安全控制措施的制定和实施提供依据，满足等保三级对"资产管理"控制点的要求。

### 第二条 依据

1. GB/T 22239-2019《信息安全技术 网络安全等级保护基本要求》资产安全要求；
2. GB/T 20984-2022《信息安全技术 信息安全风险评估方法》；
3. 《数据安全法》第二十一条数据分类分级制度；
4. 《个人信息保护法》关于个人信息处理的相关规定；
5. 本公司《信息安全管理办法》（TXK-SEC-001）。

### 第三条 适用范围

本清单覆盖屯象OS SaaS平台运营所涉及的所有重要信息资产，包括数据资产、软件资产、硬件/云资产、服务资产。

## 第二章 数据分类分级说明

### 第四条 数据分级定义

屯象OS数据资产按照安全影响程度分为以下三个级别：

**机密级（Confidential）**

定义：未授权访问或泄露可能导致以下任一后果的数据：
- 严重法律责任（违反《个人信息保护法》、《数据安全法》等）；
- 重大经济损失（影响客户财务安全或公司核心竞争力）；
- 重大声誉损失（引起媒体曝光或客户信任危机）；
- 监管处罚（罚款、吊销许可证等）。

保护要求：强制加密存储（AES-256）、加密传输（TLS 1.2+）、严格访问控制（最小权限）、完整审计日志、禁止拷贝到非授权环境。

**内部级（Internal）**

定义：仅供屯象科技内部人员使用的数据，未授权访问可能造成：
- 竞争情报泄露；
- 运营效率受损；
- 一般性声誉影响。

保护要求：访问控制、传输加密、不得对外发布。

**公开级（Public）**

定义：可对外公开发布的数据，泄露不造成实质性损害。

保护要求：完整性保护（防止篡改）、可用性保护。

### 第五条 个人信息子分类

属于机密级的个人信息（PII），按照《个人信息保护法》进一步细分：

| 子类别 | 数据示例 | 额外保护要求 |
|-------|---------|------------|
| 敏感个人信息 | 身份证号、银行卡号、生物特征、位置轨迹 | 单独同意、脱敏展示、最小化收集 |
| 一般个人信息 | 姓名、手机号、生日、消费记录 | 告知同意、访问控制 |

## 第四章 资产保护要求详细说明

### 第九条 机密级数据保护要求

**DA-001 客户会员数据（PII）**

| 保护维度 | 具体要求 |
|---------|---------|
| 数据最小化 | 仅收集业务必需的个人信息字段，禁止过度收集 |
| 存储加密 | PostgreSQL层面启用TDE（透明数据加密），手机号等字段应用层加密存储 |
| 传输加密 | 所有API通信强制TLS 1.2+ |
| 访问控制 | RLS策略强制限制租户间数据隔离；应用层API鉴权 |
| 脱敏展示 | 手机号显示为139****8888格式；身份证号仅显示后4位 |
| 数据主体权利 | 支持客户查询、更正、删除个人信息请求（《个人信息保护法》要求）|
| 访问日志 | 所有PII访问操作均产生审计日志 |
| 禁止行为 | 禁止导出到本地开发环境；禁止用于测试（须使用脱敏数据）|

**DA-006 系统配置（API密钥/JWT密钥）**

| 保护维度 | 具体要求 |
|---------|---------|
| 存储位置 | 腾讯云KMS（生产）/ 环境变量（次选，通过systemd/Docker secrets注入）|
| 禁止行为 | 严禁提交至任何Git仓库（包括私有仓库）；严禁明文记录在文档/日志中 |
| 轮换周期 | 所有密钥90天强制轮换，泄露后立即轮换 |
| 使用审计 | KMS密钥调用记录通过CloudAudit留存 |
| 访问控制 | 仅生产环境应用服务账号有KMS解密权限 |

**DA-008 审计日志**

| 保护维度 | 具体要求 |
|---------|---------|
| 写入模式 | Append-Only，禁止任何账号执行DELETE/UPDATE操作 |
| 访问控制 | 仅审计管理员账号有查询权限；系统管理员无查询权限 |
| 完整性 | CLS日志内置防篡改机制，定期哈希校验 |
| 保留期 | 强制保留180天（等保三级要求）|
| 内容要求 | 须记录：操作时间、操作账号、操作类型、操作对象、来源IP、操作结果 |

### 第十条 内部级数据保护要求

**DA-007 源代码**

| 保护维度 | 具体要求 |
|---------|---------|
| 存储位置 | GitHub私有仓库（必须），禁止在公开仓库存储 |
| 访问控制 | 仅授权开发人员有访问权限；外部人员须签署保密协议后临时授权 |
| 密钥防护 | git-secrets扫描，禁止密钥提交；CI/CD中的密钥通过GitHub Secrets管理 |
| 代码审查 | 所有合并须经过代码审查（自评或AI辅助）|
| 离职处置 | 人员离职后立即撤销GitHub访问权限 |

**DA-009 运维操作日志**

| 保护维度 | 具体要求 |
|---------|---------|
| 生成方式 | JumpServer自动录屏，人员无法关闭或删除 |
| 加密存储 | 录屏文件AES-256加密后存储于COS |
| 访问控制 | 仅审计管理员有录屏回放权限 |
| 保留期 | 强制保留180天，合规要求保留至1年 |

## 第五章 资产生命周期管理

### 第十一条 资产变更管理

1. 新增重要信息资产时，须在本清单中登记，明确分级和保护要求；
2. 资产迁移或删除前，须评估安全影响，重要资产须提交变更申请；
3. 本清单每季度审查一次，确保与实际资产状态一致；
4. 年度安全审计时，对本清单进行全面盘点。

### 第十二条 数据销毁要求

| 数据级别 | 销毁方式 | 销毁记录要求 |
|---------|---------|------------|
| 机密级 | 密码学擦除（密钥删除）+ 物理销毁（如适用）| 须提交销毁申请，留存销毁证明 |
| 内部级 | 安全删除（覆写）| 记录销毁操作日志 |
| 公开级 | 常规删除 | 无额外要求 |

腾讯云CVM/CDB等云资源退还时，须先执行密码学擦除（删除KMS密钥），再通过腾讯云官方退款流程退还，确保腾讯云存储的数据无法被后续用户恢复。
