### 场景 1：公域获客 — 微信支付"摇优惠"引流

**用户旅程**：顾客在附近门店微信支付后 → 摇一摇 → 获得屯象商家优惠券 → 跳转小程序领券 → 到店消费核销

**屯象OS对接方案**：

```
微信支付后页面
    │
    ├─→ 摇优惠/附近优惠 ──→ 跳转屯象小程序
    │                              │
    │                      ┌───────┴───────┐
    │                      │  领券 + 会员注册  │
    │                      │  (tx-member)    │
    │                      └───────┬───────┘
    │                              │
    ├─→ 商家名片 ──→ 品牌展示 + 一键关注公众号/视频号
    │
    └─→ 投放计划 API ──→ tx-growth 自动创建营销活动
```

**关键技术点**：
- 对接微信支付「投放计划」API，在 tx-growth 中自动创建/管理优惠券投放
- 支付后流量数据回传 tx-analytics，实时计算获客ROI
- 领券用户自动写入 tx-member CDP，打标签"微信支付引流"

**数据闭环**：
```
曝光 → 领券 → 到店 → 核销 → 复购
  │       │      │      │      │
  └───────┴──────┴──────┴──────┘
            全链路归因 (tx-analytics)
```

### 5.2 屯象OS核心差异化

1. **AI Native，不是AI外挂**：Agent深度嵌入每个业务环节（折扣/排菜/出餐/会员/库存/财务），而非事后分析
2. **边缘+云端双层推理**：Mac mini M4 本地推理保障实时性和离线可用，云端混元处理复杂决策
3. **Ontology统一数据模型**：微信生态各触点的用户数据统一映射到 Golden ID，避免数据孤岛
4. **事件溯源架构**：所有业务动作 = 不可变事件，微信支付回调、企微对话、会员行为全部进入事件总线
5. **三项硬约束**：毛利底线 + 食安合规 + 客户体验，AI决策不与真实经营冲突

```
    │
    ├── 小程序(Mini Program) ── miniapp-customer (顾客端)
    │                         ── miniapp-crew (员工端)
    │
    ├── 公众号(Official Account) ── 屯象科技服务号
    │
    └── 企业微信(WeCom) ── 屯象科技企业微信
```

**关键规则**：所有应用绑定到同一微信开放平台账号后，同一用户的 UnionID 唯一，实现跨应用统一身份。
