### 各阶段里程碑

| 阶段 | 核心交付 | 完成标准 |
|------|---------|---------|
| Phase 1: CI 底座 | GitHub → Harness CI → Harbor | PR 触发 CI，镜像推送成功 |
| Phase 2: CD 多环境 | CI 产物 → 6个环境自动/手动部署 | dev 自动，test/uat/pilot/prod 手动触发 |
| Phase 3: 审批+回滚 | 人工审批门禁 + 自动回滚 | prod 发布必须审批，失败自动回滚 |
| Phase 4: FF+Agent | Feature Flag 灰度 + tx-brain 接入 | Flag 变更30s内全量生效，Agent 可灰度发布 |
| Phase 5: 成本治理 | CCM 停启 + 预算 + OPA + 操作手册 | 非生产环境自动停机，成本可观测 |

## 五、Agent 体系三层解耦说明

屯象OS 的 tx-brain Agent 体系通过三层解耦实现灵活的模型切换和灰度发布：

```
┌────────────────────────────────────────────────────────────────────┐
│  Layer 1: 业务意图层（不感知具体模型）                               │
│                                                                    │
│  • AgentRouter（意图识别 + 路由决策）                               │
│  • 接收业务请求，输出标准化的 AgentTask                             │
│  • 完全通过 Feature Flag 控制路由策略                              │
│                                                                    │
│  示例: ff_multi_model_routing = "deepseek"                         │
│    → 路由到 DeepSeek V3                                            │
│  示例: ff_multi_model_routing = "claude"                           │
│    → 路由到 Claude Sonnet（高优先级任务）                           │
└──────────────────────────────┬─────────────────────────────────────┘
                               │
                               ▼
┌────────────────────────────────────────────────────────────────────┐
│  Layer 2: 模型适配层（标准化接口）                                   │
│                                                                    │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐             │
│  │ DeepSeek V3  │  │ Claude 3.5   │  │ Qwen-Long    │             │
│  │ Adapter      │  │ Adapter      │  │ Adapter      │             │
│  └──────────────┘  └──────────────┘  └──────────────┘             │
│                                                                    │
│  • 统一输入/输出格式（BaseAgent Protocol）                          │
│  • 各 Adapter 独立打包为 Docker 镜像（分开发布）                    │
│  • 通过 Harness CD 独立灰度发布各 Adapter 版本                     │
└──────────────────────────────┬─────────────────────────────────────┘
                               │
                               ▼
┌────────────────────────────────────────────────────────────────────┐
│  Layer 3: 能力执行层（Tool / 工具调用）                              │
│                                                                    │
│  ┌────────────┐  ┌────────────┐  ┌────────────┐  ┌─────────────┐  │
│  │ Menu Tool  │  │ Order Tool │  │Report Tool │  │ Search Tool │  │
│  └────────────┘  └────────────┘  └────────────┘  └─────────────┘  │
│                                                                    │
│  • 工具接口稳定，独立于模型层                                       │
│  • 工具可通过 Feature Flag 独立灰度（如 ff_agent_dish_recommend）   │
│  • 工具版本通过 Harness CD 独立发布                                │
└────────────────────────────────────────────────────────────────────┘
```

**解耦的核心价值**:
1. **模型切换不重启服务**: 通过 Flag 切换模型，30秒内生效
2. **Agent 能力独立发布**: 新增一个 Tool 不影响其他 Tool 的稳定性
3. **成本控制**: 低优先级任务路由到便宜模型，高优先级路由到高质量模型

### 环境间数据策略

| 环境 | 数据库 | 数据来源 | 数据清理 |
|------|--------|---------|---------|
| Local | SQLite 或本地 PG | 手动种子数据 | 随时重置 |
| Dev | 独立 PG Schema | CI 每次 seed | 每次部署前重置 |
| Test | 独立 PG Schema | 自动化 fixture | 测试后自动清理 |
| UAT | 独立 PG Schema | 仿真业务数据 | 手动维护 |
| Pilot | 独立 PG | **真实商户数据** | 生产级备份 |
| Prod | 独立 PG + 主从 | **真实商户数据** | 生产级备份 |
| Demo | 独立 PG Schema/Tenant | 演示种子数据 | 7天后自动清理 |

## 七、成本治理策略

```
月度预算: ¥3,000
          │
          ├── Prod (60%):  ¥1,800/月  ← 7x24必须保障
          ├── Pilot (20%): ¥600/月   ← 7x24必须保障
          └── 非生产 (20%): ¥600/月  ← 通过AutoStopping压缩实际费用
                    │
                    ├── Dev:   夜间停机（22:00-08:30），节省约60%
                    ├── Test:  周末全停，节省约30%
                    ├── UAT:   4h无活动停，节省约50%
                    └── Demo:  7天TTL，按需创建

目标: 非生产实际花费 < ¥300/月（预算¥600，通过AutoStopping节省50%+）

LLM API 专项预算: ¥500/月
          ├── tx-brain 主推理: ¥250
          ├── Agent 实验: ¥150（GPU实例4h上限）
          └── 其他辅助: ¥100

总成本目标: < ¥2,700/月（低于¥3,000预算10%缓冲）
```

### 成本可观测能力

1. **Harness CCM Dashboard** — 实时查看各环境成本趋势
2. **月度成本报告脚本** — `scripts/cost-report.sh` 每月自动生成 Markdown 报告
3. **预算告警** — 80%/100% 两级告警，分别发企微和电话
4. **AutoStopping 日志** — 记录每次自动停机节省的费用
