## 二、DEMO 路径（10 步）

> 演示总时长 12-15 分钟。复购率/会员深度是 zqx 主线。

| # | 操作 | 预期结果 |
|---|---|---|
| 1 | 打开安卓 POS（商米 T2），登录 | 桌台图加载，左侧 Agent 预警栏空 |
| 2 | 收银员扫顾客会员二维码识别 | 拉到顾客 Golden ID + RFM 标签：「沉睡-高价值-本周首次回访」 |
| 3 | 服务员 PWA 同步看到顾客标签，建议口味（辣度/忌口） | web-crew 显示 AI 卡片：「该顾客上次点了酸汤鱼，推荐折耳根炒腊肉」 |
| 4 | 点单：2 主菜 + 1 小吃 + 2 饮料 | 订单写入 tx-trade，KDS 实时收到 |
| 5 | 收银台请求会员日 9 折 | 折扣守护 Agent 校验：复购券+会员日叠加，毛利 18.5% > 阈值，放行 |
| 6 | 微信支付 → ORDER.PAID + DISCOUNT.APPLIED 事件 | 物化视图 mv_member_clv 异步更新该顾客 LTV |
| 7 | 切到 web-admin "会员洞察" 页（**会员洞察 Agent**） | 显示当月新增会员、沉睡唤醒数、客单价同比 |
| 8 | 触发"沉睡会员券推送"演示（私域运营 Agent） | 50 条短信任务进队列，模拟 1 条发送成功（不实发） |
| 9 | 切到"复购率"看板 | 30 天复购率 28% → 目标 33%，趋势可见 |
| 10 | reset 脚本清演示数据 | 环境复位 |

## 四、演示前 60 秒话术（现场用）

> "最黔线在长沙做黔菜很多年，老客户基础不错。痛点是——你们说得出哪些是老客户，但说不出哪些老客户已经流失、流失了多久、流失的原因是什么。
>
> 屯象OS的会员引擎做三件事：第一，给每个顾客建一个 Golden ID，所有渠道的消费数据合并；第二，按 RFM 自动打标签，告诉服务员这位顾客上次什么时候来、点了什么、辣度偏好；第三，当顾客进入'沉睡'状态时，自动触发一张唤醒券。
>
> 我们的目标不是给你一个会员系统，是把你们的复购率从 28% 提到 33%。这个 5 个百分点，就是凭空多出来一批老顾客。"

## 五、回退方案

| 场景 | 紧急动作 |
|---|---|
| Golden ID 识别不到 | 跳过第 2 步，话术过渡："今天用未注册顾客演示，注册流程后台一样" |
| 沉睡会员券任务失败 | `flags/agents/private_ops` 关 → 跳过第 8 步 |
| RLS smoke 测红 | **立刻取消会员洞察模块演示**，只演 POS 收银主线 |
| 适配器同步报错 | 切手动 mock 数据；脚本：`python3 scripts/seed_demo.py --merchant zqx --mock-only` |
| 整体环境炸了 | `bash scripts/demo-reset.sh && python3 scripts/seed_demo.py --merchant zqx --check` |
